
- 第四课你学会了继承——子类自动获得父类的代码。但继承有一个问题：父类的方法必须写具体实现，哪怕这个实现毫无意义。抽象类和接口就是来解决“父类不知道该怎么实现，但子类必须实现”的场景。

---

### 一、抽象类

**问题场景：** `Person` 的 `introduce()` 方法——不同职业的人介绍方式完全不同，在 `Person` 里写任何具体实现都是错的。最好的方案是：只声明“所有 Person 子类必须能 introduce”，但具体怎么实现，子类自己决定。

**解决方案：抽象类 + 抽象方法**


**关键点：**
- `abstract class` 不能 `new `，它只能被继承
- 抽象方法只有签名，没有大括号和实现，分号直接结束
- 抽象方法必须在子类中被重写，否则子类也必须是抽象类
- 有抽象方法的类必须声明为 `abstract class`

### 二、接口（Interface）

抽象类还是一个类，只能单继承。接口不是类，是一个“能力契约”——只定义“能做什么”，不定义“怎么做”，一组**方法签名的集合**，

**现实类比：** “会飞”是一种能力。鸟会飞，飞机也会飞，但鸟和飞机没有继承关系。接口就是声明“会飞”这个能力，不管你是谁，只要实现了这个接口，就具备这个能力。

**语法：**

```java
public interface Flyable {
    void fly();    // 接口中的方法默认是 public abstract，可以省略
}
```

**语法规则**：
- 用 `interface` 关键字声明
- 接口中的方法**默认是 `public abstract`**，不用写
- 接口中不能有构造器，不能有实例方法
- 接口不能直接 `new`

- 类用 `implements` 实现接口，承诺自己会实现接口中所有方法：

**关键规则**：
- 实现类**必须**重写接口中所有方法，否则必须声明为抽象类
- 一个类只能继承一个父类，但可以**实现多个接口**
- 多个无关的类可以实现同一个接口
- 接口可以当作变量类型使用，**JVM 的底层机制和继承多态完全一样——都是动态绑定**

---

**接口与抽象类的核心区别：**

| 对比维度  |        抽象类         |            接口             |
| :---: | :----------------: | :-----------------------: |
| 回答的问题 |   **是什么**（父子关系）    |      **能做什么**（能力契约）       |
|  关键字  |  `abstract class`  |        `interface`        |
| 继承/实现 | 子类 `extends`（只能一个） |  实现类 `implements`（可以多个）   |
| 构造方法  |         有          |            没有             |
| 成员变量  |      可以有普通变量       |          只能有静态常量          |
|  方法   |    可以有抽象方法和普通方法    | 默认全是抽象方法（Java 8+ 可以有默认方法） |
|  属性   |      可以有实例变量       |           只能有常量           |
|  本质   |       “是什么”        |          “能做什么”           |

---

**一个类可以实现多个接口：**

```java
public class Son extends Father implements Flyable, Swimmable, Runnable {
    // 必须实现所有接口的所有方法
}
```

这是 Java 解决“多继承”的方式——类只能继承一个父类，但可以实现无数个接口。

- 两个接口有同名且签名相同的方法 → 不冲突，类写一个就行
- 两个接口有同名但签名不同的方法 → 冲突，**类无法同时实现**
