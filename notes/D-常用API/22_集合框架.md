# ArrayList —— 动态数组

数组一旦创建，长度就固定了。`ArrayList` 是 Java 提供的动态数组，容量会自动增长，是日常开发中最常用的数据结构。

```java
import java.util.ArrayList;

public class ArrayListDemo {
    public static void main(String[] args) {
        // 创建 ArrayList，尖括号里指定存放的类型
        ArrayList<String> list = new ArrayList<>();
        
        // 1. 添加元素
        list.add("Zoe");
        list.add("Tom");
        list.add("Jerry");
        System.out.println("列表: " + list);
        
        // 2. 获取元素
        System.out.println("第0个: " + list.get(0));
        
        // 3. 大小
        System.out.println("大小: " + list.size());
        
        // 4. 修改元素
        list.set(1, "Tommy");
        System.out.println("修改后: " + list);
        
        // 5. 删除元素
        list.remove(0);
        System.out.println("删除后: " + list);
        
        // 6. 判断是否包含
        System.out.println("包含Jerry? " + list.contains("Jerry"));
        
        // 7. 遍历——可以用普通 for 循环
        for (int i = 0; i < list.size(); i++) {
            System.out.println("遍历: " + list.get(i));
        }
        
        // 8. 增强 for 循环（专门用来遍历集合）
        for (String name : list) {
            System.out.println("增强遍历: " + name);
        }
        
        // 9. 存储基本类型需要包装类
        ArrayList<Integer> scores = new ArrayList<>();
        scores.add(95);
        scores.add(88);
        scores.add(76);
        System.out.println("成绩列表: " + scores);
    }
}
```


## 常用方法

| 方法 | 作用 |
|------|------|
| `add(元素)` | 在末尾添加 |
| `get(索引)` | 获取指定位置元素 |
| `set(索引, 元素)` | 修改指定位置元素 |
| `remove(索引)` | 删除指定位置元素 |
| `size()` | 返回元素个数 |
| `contains(元素)` | 判断是否包含 |

## 遍历方式

```java
// 方式一：普通 for
for (int i = 0; i < list.size(); i++) { }

// 方式二：增强 for（推荐）
for (类型 变量 : 集合) { }
```

## 存储基本类型

ArrayList 不能直接存储 `int`、`double` 等基本类型，需要用包装类：`Integer`、`Double`、`Boolean` 等。

# 集合

## Collection接口

1. 集合的定义：集合是一种引用数据类型，用于存储一组元素并提供了多种数据结构供我们使用，相比数组，集合对其中的元素管理更加便捷。
    
2. **`Collection（集合）`接口定义：**`java.util.Collection`接口是所有单列集合的顶级接口，定义了所有单列（一个元素由一个数据组成）集合的相关操作方法。
    
    - 常见子接口
        
        - `List`：有序，可重复，有下标可操作（一定有序）。
            
        - `Set`：有的有序，有的无序，不可重复，无下标不可操作。
            
    - 集合的应用率大于数组。
        
3. **`Collection`集合的使用：**
    
    - 集合可以伸缩，不用在声明时指定长度
        
    - `.add(Object o)`，增加元素
        
    - `.size()`，集合大小，输出集合的元素个数
        
    - `.isEmpty()`，是否是空集（`size`==0时表示空集）
        
    - `.remove(Object o)`，从当前集合中删除与给定元素`o`使用`equals()`比较为`true`的元素。
        
        - 若存在重复元素则只删除第一次匹配的。
            
    - `.clear()`，清空集合中的所有元素
        
    - `.contains(Object o)`判断当前集合是否包含给定元素
        
        - 判断的依据是给定元素与当前集合元素是否存在`equals()`为`true`的情况，因此重写了`equals()`之后进行判断，就不在乎给定元素是不是装进集合了，而只对内容进行比较。
            
    - `.iterator()`，获取迭代器进行遍历
        
        - 这个方法是`Collection`给我们提供的一个统一的遍历集合的方式：迭代器模式。通过集合的`.iterator()`方法可以获取一个遍历当前集合的迭代器（`Iterator`接口）
            
    - 集合中的元素可以直接输出不用遍历形式输出，因为重写了`toString`。返回格式如下：
        
        [元素1.toString(), 元素2.toString(), 元素3.toString(), 元素4.toString(), ········]
        
    - 集合中存储的都是引用数据类型，引用里面存储的是对象的地址指向对象，因此集合中存储的是元素指向的对象的地址。当对象属性变化的时候，集合中对应的元素的属性也会发生变化。

### 迭代器

1. **迭代器定义：**Java迭代器（Iterator）是 Java 集合框架中的一种机制，是一种用于遍历集合（如列表、集合和映射等）的接口。它提供了一种统一的方式来访问集合中的元素，不需要了解底层集合的具体实现细节。
    
    - 不是所有集合都有下标，只有`List`集合有下标，`Set`集合没有下标，因此无法直接使用`for`循环的方式遍历所有`Collection`集合。通过使用迭代器，我们可以逐个访问每个集合中的元素，这种方法适用于各种类型的集合。
        
    - `Iterator`这个接口定义了迭代器遍历集合的相关操作，不同集合提供不同的迭代器实现类，我们不需要记住，只需要从多态的角度看待就可以了。
        
2. **迭代器的常用方法：**迭代器遍历遵循的步骤为：使用`while`循环，问`hasNext`，取`next`，删`remove`----删并非必要操作
    
    - Iterator 类位于 java.util 包中，使用前需要导入它，语法格式如下：
        
        import java.util.Iterator; // 引入 Iterator 类
        
    - `.hasNext()` - 用于判断集合中是否还有下一个元素可以访问。
        
        - 迭代器的起始位置在集合第一个元素之前
            
    - `.next()` - 返回迭代器的下一个元素，并将迭代器的指针移到下一个位置。
        
    - `.remove()` - 从集合中删除迭代器最后访问的元素（可选操作）。
        
        - 迭代器遍历过程中绝对不允许通过_集合_的方式来增删元素，否则会导致 ConcurrentModificationException 异常，可以通过迭代器自身的`remove()`方法，删除`next()`方法获取的那个元素。
            
3. **迭代器执行过程详解：**![在这里插入图片描述](https://i-blog.csdnimg.cn/blog_migrate/25c06405128bdd89361a8995762dc65f.gif)
    
    - ①首先得到一个集合的迭代器Iterator iterator = list.iterator();
        
    - ②进入while循环，调用hasNext()判断是否有下一个元素，返回true，Iterator.next()移动一个位置，将该位置的元素111返回。
        
    - ③再次进入while循环，调用hasNext()判断是否有下一个元素，返回true，Iterator.next()移动一个位置，将该位置222的元素返回。
        
    - ④再次进入while循环，调用hasNext()判断是否有下一个元素，返回true，Iterator.next()移动一个位置，将该位置333的元素返回。
        
    - ⑤再次进入while循环，调用hasNext()判断是否有下一个元素，返回false，循环结束。
        
    - 在 迭代器的遍历过程中先通过hastNext()方法判断是否有下一个元素，如果存在下一个元素再调用next()方法获取元素，在这里next()方法先往后移动一个元素位置，再返回该位置的元素。因此，在调用next()方法之前必须要调用hastNext()方法进行检测；如果没有调用并且没有下一个元素，直接调用next()方法会抛出 NoSuchElementException异常。

### 增强`For`循环

1. **增强`For`循环的定义：**增强`For`循环是`JDK1.5`时推出的，用来代替`Iterator`迭代器，可以把它看做简化版的`Iterator`，其实它的底层实现就是`Iterator`迭代器，只能用于遍历集合或数组。
    
    - 增强`For`循环，也称为新循环，因为它的内部还是通过迭代器实现的，所以不能动态增删元素，只能做遍历。
        
    - 新循环不是新的语法，而是编译器帮我补充代码，数组还是`For`循环，集合还是迭代器循环。
        
2. **增强`For`循环的语法：**
    
    for (Type element : collection) {  
        // 循环体：对 element 进行操作  
    }
    
    - **Type：**表示集合或数组中元素的类型。
        
    - **element：**一个临时变量，在每次迭代中自动接收集合或数组中的当前元素。
        
    - **collection：**要遍历的目标，可以是数组或实现了 `Iterable` 接口的集合。
        
    - **:：**分隔符，读作“in”，表示从 `collection`中逐一取出元素。
        
    - 在循环执行时，`element` 会依次被赋值为 `collection` 中的每个元素，从头到尾自动完成遍历，无需显式控制循环条件。
        
3. **增强`For`循环的快捷键：**与迭代器一样，增强`For`循环也有快捷键，输入`I`回车即可快速生成。

### `List`集合

1. **`List`集合的定义：**继承自`Collection`接口，以线性方式存储元素。`List`是有序(存取一致)、可重复、有索引的集合
    
    - `List`的两个常见的实现类：`ArrayList`和`LinkedList`，分别用动态数组和链表的方式实现了`List`接口。
        
2. **`ArrayList`和`LinkedList`两者的区别（是一个常见面试题）：**
    
    - `ArrayList`内部使用数组实现，代表长度可变的数组，动态数组，初始长度为10，装满时自动扩容至之前的1.5倍。里面的每个元素都有一个物理地址，直接根据地址(根据下标找地址)找到数据，找每个元素的效率都是一样的快，增删数据效率较低。
        
        - 查询性能更好，增删性能不好
            
    - `LinkedList`内部使用链表实现，特点是将数据封装到一个一个的节点中`noted`，只记首尾，将其他节点按顺序连接形成一条链，一个节点由三个部分组成，第一部分存储的是上一个元素的地址，第二部分存储的是本身的数据，第三部分存储的是下一个元素的地址。`LinkedList`自身有两个节点头和尾。所以链表只能直接找到头尾的数据，头尾的访问效率高，访问中间元素效率低。增删时只要把元素前后的节点地址改了就行，增删效率较高，但还是要先找到要删的元素在这方面效率不高，所以只在首尾的查找增删效率都很高。
        
        - 两端效率高，中间效率低
            
    - 仅当两端操作数据时才选择`LinkedList`其他都是`ArrayList`
        
3. **`ArrayList`和`LinkedList`的使用方法：**`ArrayList`和`LinkedList`实现了`LIst`接口，而`List`接口继承自`Collection`接口，因此`Collection`本身的方法，`ArrayList`和`LinkedList`也可以使用。而`ArrayList`和`LinkedList`虽然底层存储方式不同，但使用是一模一样的。
    
    - import java.util.ArrayList; // 引入 ArrayList 类  
          
        ArrayList<E> objectName =new ArrayList<>();　 // 初始化  
        //E: 泛型数据类型，用于设置 objectName 的数据类型，只能为引用数据类型。  
        //objectName: 对象名。
        
    - import java.util.LinkedList; // 引入 LinkedList 类  
          
        LinkedList<E> list = new LinkedList<E>();   // 普通创建方法  
        或者  
        LinkedList<E> list = new LinkedList(Collection  《? extends E> c); // 使用集合创建链表
        
    - `.get(int index)`方法，获取下标位置的元素并返回
        
    - `.set(int inddet,Object o)`方法，将下标位置的元素改为给定的元素并返回原始元素
        
    - `.add(Object o)`方法，增加元素
        
        - 重载之后的`.add(int index,Object o)`方法是在下标位置和下标前一个位置的中间插入给定元素。如果使用重载的`.add(int index,Object o)`方法指定下标为最后一个元素的下一个位置时，就相当于追加元素，如歌下标大于最后一个元素的下一个位置时，则直接报异常。
            
    - `.remove(Object o)`方法，从当前集合中删除与给定元素`o`使用`equals()`比较为`true`的元素。
        
        - 重载之后的`.remove(int index)`方法，可以删除下标位置的元素，并返回其删除的元素。
            
4. **`LinkedList`的特有方法：**以上的方法`LinkedList`也可以使用，下面这些方法是针对`LinkedList`特有的属性所重载的一些方法，都是针对首尾所作的操作。
    
    - `.addFirst`方法，在头部增加元素
        
    - `.addLast`方法，在尾部增加元素-0
        
    - `.getLast`方法，获取尾部元素
        
    - `.removeFirst`方法，删除头部元素
        
    - `.removeLast`方法，删除尾部元素
        

### `Set`集合

1. **`Set`集合的定义：**继承自`Collection`接口，`Set`是有的有序，有的无序，不可重复，无索引的集合。
    
    - `Set`接口有两个常见的实现类：
        
        - `HashSet`：由无序（存取不一致），不可重复，无索引的集合
            
        - `LinkedHashSet`：有序（存取一致），不可重复，无索引的集合-----且继承自`HashSet`
            
        - `HashSet`比`LinkedHashSet`使用频率高
            
2. **`HashSet`和`LinkedHashSet`的常用方法：**`Set`接口继承自`Collection`接口，因此`Collection`接口的方法，`HashSet`也可以用
    
    - `.add()`方法，增加元素，如果添加成功则返回true，添加失败则返回false。因为Set是不重复的集合，因此能不能添加看的是`equals()`方法和`hashcode()`方法是否相等。（重写这俩个方法的重要性）
        
    
    只要名字有hash的都和hashcode有关
    

### 集合排序

1. Collections类是集合的工具类，包含了很多操作集合的静态方法。其中对集合排序的方法已经做好了我们只需要用就就行
    
2. 其中两个静态方法：
    
    - Collections.sort(List list)方法：可以对List集合进行自然排序(从小到大)（数字从小到大，字符（中文和英文）按照ASCILL码排序），但只能排序集合中实现了Comparable接口的元素，若没有实现去排序则会报异常。
        
    - 一般我们自己写的类不去实现这个接口，因为会对我们的程序有侵入性。
        
        - 侵入性:要求为其增加或者修改其他额外代码。侵入性越强越不利于后期维护
            
    - 因此，当我们想排序一个元素没有实现Comparable接口的List集合时，建议调用Collections.sort的重载方法：Collections.sort(List list, Comparator o);该方法可以通过比较器来自定义比较规则。
        
        - compare()方法用于自定义比较规则，通过返回值来表达大小关系
            
        - 返回值实现的要求:----------不用纠结，记住下面的结论即可
            
            1)如果返回值>0则表达的是o1>o2；
            
            2)如果返回值<0则表达的是o1<o2；
            
            3)如果返回值==0则表达的是o1=o2；
            
        - 结论:1)前面的-后面的-------------升序；2)后面的-前面的-------------降序；
            
    - Collection.reverse()方法：反转集合
        

## Map接口

1. **`Map（集合）`接口定义：**是所有双列集合的顶级接口，定义了所有双列集合的相关操作方法。
    
    - Map体现的结构是一个多行两列的表格，总是成对保存数据，每一对称为一个”Entry“，左列为key，右列为value。要求key可以重复但是value不可重复（判断依据为equals()+hashCode()）
        
    - 常用实现类：
        
        1. `HashMap`：内部由hash表实现，是无序，不可重复，无索引的集合。
            
        2. `LinkedHashMap`：内部由链表维护顺序实现的，是有序，不可重复，无索引的集合-------继承自HashMap
            
    - 集合的应用率大于数组。
        
2. **Map常用方法：**
    
    - put()：添加元素
        
        - put(K k,V v):向map中添加一组键值对
            
        - Map要求key不能重复，如果使用重复的key存value时实际是做替换value的操作，同时将被替换的value通过返回值返回，如果是第一次存入，则返回值为null
            
    - 以下方法与Collection接口方法一致
        
        - get()：根据键获取值
            
        - size()：获取集合大小
            
        - isEmpty()：判断集合是否是空集
            
        - containsKey()：判断集合是否包含键key
            
        - containsValue()：判断集合是否包含值value
            
        - remove()：根据键删除键值对，并返回被删除的value值
            
        - clear()：清空集合
            
3. **Map支持三种遍历方式:**
    
    1. 遍历所有的key：ketSet()方法，返回一个Set集合
        
    2. 遍历所有的value(相对不常用)：values()方法，返回一个Collention集合
        
    3. 遍历所有的键值对：entrySet()方法，返回一个Set集合
        
    4. forEach()：基于lambda表达式遍历集合
        

### LinkedHashMap

`LinkedHashMap`：内部由链表维护顺序实现的，是有序，不可重复，无索引的集合-------继承自HashMap，操作方法上与Map一致，不同在于创建时的构造函数有两个：

1）.无参构造：默认不记录访问顺序

2）.有参构造：三个参数，int类型的初始容量，float类型的扩容因子，boolean类型的是否记录访问顺序
