简述字节流与字符流的区别。**

- 字节流：以字节（8位）为单位，抽象基类为`InputStream`/`OutputStream`，适合所有文件类型。  
- 字符流：以字符（16位Unicode）为单位，抽象基类为`Reader`/`Writer`，内部自动处理编码，适合文本文件。  
- 字符流 = 字节流 + 编码表。


简述序列化的概念及实现条件。**

序列化：将对象的状态转换成字节序列以便存储或网络传输。  
实现条件：  
① 类必须实现`Serializable`接口（标记接口，无方法需实现）。  
② 用`ObjectOutputStream`写入，`ObjectInputStream`读取。  
③ `transient`修饰的字段不被序列化，反序列化后恢复默认值。  
④ 建议显式定义`serialVersionUID`保证版本兼容性。




简述`sleep()`和`wait()`的区别。**

| `sleep()`      | `wait()`                    |
| -------------- | --------------------------- |
| `Thread`类的静态方法 | `Object`类的方法                |
| 任何地方都可调用       | 必须在`synchronized`块中调用       |
| 休眠时**不释放锁**    | 等待时**释放锁**                  |
| 时间到自动唤醒        | 需`notify()`/`notifyAll()`唤醒 |
|                |                             |




简述死锁产生的四个必要条件。**

① 互斥条件：资源只能被一个线程独占。  
② 请求与保持：持有一个资源的同时请求另一个。  
③ 不可剥夺：资源不能被强行抢走。  
④ 循环等待：形成线程-资源的等待环路。  
打破任一条件可避免死锁，常用方法：按序加锁。





简述线程的生命周期状态。**

① 新建：`new Thread()`创建线程对象。  
② 就绪：调用`start()`后，等待CPU调度。  
③ 运行：获得CPU时间片，执行`run()`方法。  
④ 阻塞：`sleep()`、`wait()`、`join()`或等锁时进入，阻塞结束回到就绪。  
⑤ 死亡：`run()`执行完毕或异常退出。




**简要叙述线程死亡的两个原因。**  

run()方法执行完毕；线程内抛出未捕获异常导致终止

# Java IO

## （一）IO全解析

1. 在 Java 编程的广阔天地里，Java IO（Input/Output）就如同一个关键的纽带，连接着程序与外部世界 。无论是日常开发里频繁的文件读取和写入，还是构建网络应用时实现客户端与服务器之间的数据交互，又或是进行数据持久化存储，Java IO 都发挥着无可替代的作用。可以毫不夸张地说，只要涉及到数据的输入与输出，Java IO 必然参与其中。它就像一座桥梁，让程序能够与外部的各种数据源进行顺畅的沟通，为丰富多样的应用场景奠定了坚实的基础，因此深入理解 Java IO 是每一个 Java 开发者的必修课。
    
2. **什么是IO：**IO，即Input和Output，输入与输出。输入就是程序从外部获取数据，输出就是程序向外界发送数据
    
3. **什么是流：**在Java IO中，流（Stream）的概念十分重要。可以把流想象成一条数据传输的通道，数据就像水流一样在这个通道中流动。它具有方向性，数据从数据源流向程序（输入流），或者从程序流向数据目的地（输出流）。并且，流中的数据是按顺序依次传输的，就像水流是依次流淌一样 。数据在流中是以字节（byte）或者字符（char）为单位进行传输的。
    
    > 流的读写是顺序读写的，只能顺序向后写或向后读，不能回退。
    
    1. **按照数据流的方向进行分类：**
        
        - **输入流：**用于从外部数据源读取数据到程序中。在 Java 中，所有输入流的基类是InputStream（字节输入流）和Reader（字符输入流） 。
            
        - **输出流：**用于将程序中的数据写入到外部数据目的地。Java 中所有输出流的基类是OutputStream（字节输出流）和Writer（字符输出流） 。
            
    2. **按照数据处理的单位进行分类：**
        
        - **字节流：**`InputStream`/`OutputStream`（二进制数据）。以字节（8 位）为单位处理数据，适用于处理任何类型的数据。InputStream和OutputStream是字节流的抽象基类。
            
        - **字符流：**`Reader`/`Writer`（文本数据）。以字符（16 位 Unicode）为单位处理数据，主要用于处理文本数据 。Reader和Writer是字符流的抽象基类。字符流在处理文本时，会自动进行字符编码转换，方便处理不同编码格式的文本文件。
            
        - **字节流和字符流的区别：**字节流和字符流的主要区别在于处理数据的单位和应用场景。字节流更通用，可以处理任何类型的数据，但在处理文本时需要手动处理字符编码问题；字符流专门用于处理文本数据，自动处理字符编码，使用起来更方便，但不适用于二进制数据的处理。简而言之，字节是给计算机看的，字符才是给人看的。
            
    3. **按照功能进行分类：**
        
        - **节点流：**也称为低级流，是直接与数据源或数据目的地相连的流，负责从数据源读取数据或直接将数据写入数据目的地 。
            
        - **处理流：**也称为包装流、高级流，是在节点流的基础上进行包装，增强了流的功能。处理流不能单独存在，必须连接在其他流（通常是节点流）之上。
            
    4. **按照数据操作对象进行分类：**

        |类别|字节流|字符流|
        |---|---|---|
        |**文件**|FileInputStream/FileOutputStream|FileReader/FileWriter|
        |**数组**|ByteArrayInputStream/ByteArrayOutputStream|CharArrayReader/CharArrayWriter|
        |**管道**|PipedInputStream/PipedOutputStream|PipedReader/PipedWriter|
        |**基本数据类型**|DataInputStream/DataOutputStream|-|
        |**缓冲**|BufferedInputStream/BufferedOutputStream|BufferedReader/BufferedWriter|
        |**打印**|PrintStream|PrintWriter|
        |**对象序列化**|ObjectInputStream/ObjectOutputStream|（要序列化的对象必须实现 `Serializable` 接口，否则抛出 `NotSerializableException`。）-|
        |**转换**|-|InputStreamReader/OutputStreamWriter|
        

## （二）按照数据处理的单位进行分类

### 1.字节流全解析

1. **字节流全解析：**在 Java I/O 流体系中，**所有继承自 `InputStream` 和 `OutputStream` 的子类都支持其常用方法**。![img](https://www.pdai.tech/images/io/java-io-category-1-1.png)
    
    1. **InputStream详解：**
        
        1. **InputStream 概述：**InputStream是 Java IO 中所有字节输入流的抽象父类 ，它为从各种数据源读取字节数据提供了基本的方法定义。InputStream本身是抽象类，不能直接实例化，它的子类通过实现InputStream的抽象方法，实现了针对不同数据源的具体读取操作。
            
        2. **InputStream 常用方法：**
            
            1. **`public abstract int read()`** 方法是`InputStream`中最基本的读取方法，它从输入流中读取一个字节的数据，并返回字节表示的`int`值（0~255）（字节值会被提升为int类型的低 8 位，高 24 位补零）。如果已读到末尾，返回`-1`表示不能继续读取了，从而实现逐字节读取文件内容 。
                
            2. **`public int read(byte[] b)`** 方法从输入流中读取一定数量的字节，并将其存储在缓冲区数组b中，返回int类型的实际读取的字节数 。如果数组 `b` 的长度为零，则不读取。如果返回 `-1`，表示文件结束了。
                
            3. **`public int read(byte b[], int off, int len)`** ：在`read(byte b[ ])` 方法的基础上增加了 `off` 参数（下标索引）和 `len` 参数（要读取的最大字节数）。从输入流中读取最多 `len` 个字节的数据，存储到字节数组 `b` 中，从数组的 `off` 索引位置开始写入。
                
            4. **`public long skip(long n)`** ：忽略输入流中的 n 个字节 ,返回实际忽略的字节数。
                
            5. **`public int available()`** ：返回当前输入流中可读的字节数，可以获取当前文件中还能读取的字节数量 。
                
            6. **`public void close()`** ：关闭流，并释放与该流关联的所有系统资源 。`InputStream`和`OutputStream`都是通过`close()`方法来关闭流。在使用完Stream后，必须调用close()方法，以确保系统资源的正确释放，避免资源泄漏。在try块中使用了 Java 7 引入的 “自动资源管理”（try-with-resources）语法，它会在try块结束时自动调用inputStream.close()，简化了资源关闭的操作 。
                
            7. **`public synchronized void mark(int readlimit)`**：标记读取位置，下次还可以从这里开始读取，使用前要看当前流是否支持，可以使用 markSupport() 方法判断
                
            8. **`public synchronized void reset()`**：重置读取位置为上次 mark 标记的位置
                
            9. **`public boolean markSupported()`** ：判断当前流是否支持标记流，和上面两个方法配套使用
                
            10. JDK9 新增：
                
                - **`public byte[] readAllBytes()`**：读取输入流中的所有字节，返回字节数组，调用readNBytes(Integer.MAX_VALUE)方法。
                    
                - **`public int readNBytes(byte[] b, int off, int len)`**：阻塞直到读取 `len` 个字节，此方法不会关闭输入流。
                    
                - **`public byte[] readNBytes(int len)`**：（JDK11更新）阻塞直到读取 `len` 个字节，此方法不会关闭输入流。
                    
                - **`public long transferTo(OutputStream out)`**：将所有字节从一个输入流传递到另一个输出流。
                    
    2. **OutputStream详解：**
        
        1. **OutputStream 概述：**OutputStream是 Java IO 中所有字节输出流的抽象父类 ，它为将字节数据写入各种数据目的地提供了基本的方法定义。与InputStream相对应，OutputStream同样是抽象类，不能直接实例化，它的子类通过实现OutputStream的抽象方法，实现了针对不同数据的具体写入操作。
            
        2. **OutputStream 常用方法：**
            
            1. `write(int b)`：将特定的一个字节写入输出流，用来向文件中写入1个字节。
                
            2. `write(bytep[] b)` : 将数组`b`中的所有字节一次性写入到输出流文件中，实际调用的是 `write(b, 0, b.length)`方法。
                
            3. `write(byte[] b, int off, int len)` : 在`write(byte b[ ])` 方法的基础上增加了 `off` 参数（下标索引）和 `len` 参数（要写入的字节数）。将字节数组 `b` 中从 `off` 开始的 `len` 个字节写入输出流（如文件）
                
            4. `flush()`：刷新此输出流并强制写出所有缓冲的输出字节；默认是空实现宫子类覆盖
                
            5. `close()`关闭流，并释放与该流关联的所有系统资源 。`InputStream`和`OutputStream`都是通过`close()`方法来关闭流。在使用完Stream后，必须调用close()方法，以确保系统资源的正确释放，避免资源泄漏。在try块中使用了 Java 7 引入的 “自动资源管理”（try-with-resources）语法，它会在try块结束时自动调用inputStream.close()，简化了资源关闭的操作 。
                

### 2.字符流全解析

1. **字符流全解析：**在 Java I/O 流体系中，**所有继承自 `Reader` 和 `Writer` 的子类都支持其常用方法**。![img](https://www.pdai.tech/images/io/java-io-category-2-1.png)
    
    1. **Reader详解：**
        
        1. **Reader概述：**Reader是 Java IO 中所有字符输入流的抽象父类 ，它为从各种数据源读取字符数据提供了基本的方法定义。Reader本身是抽象类，不能直接实例化，它的子类通过实现Reader的抽象方法，实现了针对不同字符数据源的具体读取操作 。与字节输入流InputStream不同，Reader是以字符（16 位 Unicode）为单位进行数据读取的，这使得它在处理文本数据时更加方便和高效 。
            
        2. **Reader的常用方法：**
            
            1. **抽象方法（这些方法是 `Reader` 类中定义的抽象方法，所有子类必须实现它们。）**
                
            2. **`abstract int read(char[] cbuf, int off, int len) throws IOException`**
                
                1. **作用**：从输入流中读取最多 `len` 个字符，存储到字符数组 `cbuf` 中，从 `off` 索引位置开始写入。
                    
                2. 参数：
                    
                    - `cbuf`：目标字符数组。
                        
                    - `off`：写入字符数组的起始索引。
                        
                    - `len`：要读取的最大字符数。
                        
                
                - **返回值**：实际读取的字符数，若已到达流末尾则返回 `-1`。
                    
            3. **`abstract int read(CharBuffer target) throws IOException`**
                
                - **作用**：将字符读入 `CharBuffer` 缓冲区。
                    
                - **参数**：`target` 是目标字符缓冲区。
                    
                - **返回值**：实际读取的字符数，若已到达流末尾则返回 `-1`。
                    
                - **注意**：`CharBuffer` 是 Java NIO 中的类，用于高效处理字符缓冲。
                    
            4. **非抽象方法（这些方法由 `Reader` 类提供默认实现，子类可以选择覆盖。）**
                
            5. **`int read() throws IOException`**
                
                - **作用**：读取单个字符，返回其 Unicode 值（0~65535）。
                    
                - **返回值**：读取的字符值（0~65535），若已到达流末尾则返回 `-1`。
                    
            6. **`int read(char[] cbuf) throws IOException`**
                
                - **作用**：将字符读入数组 `cbuf`。
                    
                - **参数**：`cbuf` 是目标字符数组。
                    
                - **返回值**：实际读取的字符数，若已到达流末尾则返回 `-1`。
                    
                - **等价于**：`read(cbuf, 0, cbuf.length)`。
                    
            7. **`void close() throws IOException`**
                
                - **作用**：关闭流并释放相关资源。
                    
                - **注意**：使用完流后必须调用 `close()`，否则可能导致资源泄漏。
                    
            8. **`long skip(long n) throws IOException`**
                
                - **作用**：跳过输入流中的 `n` 个字符。
                    
                - **参数**：`n` 是要跳过的字符数。
                    
                - **返回值**：实际跳过的字符数。
                    
                - **注意**：并非所有流都支持跳过操作（如 `FileReader` 不支持）。
                    
            9. **`boolean markSupported()`**
                
                - **作用**：判断流是否支持 `mark()` 和 `reset()` 操作。
                    
                - **返回值**：`true` 表示支持，`false` 表示不支持。
                    
            10. **`void mark(int readAheadLimit) throws IOException`**
                
                - **作用**：在流中标记当前位置，后续可通过 `reset()` 回退。
                    
                - **参数**：`readAheadLimit` 是标记后可读取的最大字符数。
                    
                - **注意**：若流不支持标记（如 `FileReader`），调用此方法会抛出异常。
                    
            11. **`void reset() throws IOException`**
                
                - **作用**：将流重置到最近一次 `mark()` 标记的位置。
                    
                - **注意**：必须在调用 `mark()` 后调用 `reset()`，否则抛出 `IOException`
                    
            12. **`boolean ready() throws IOException`**
                
                - **作用**：判断流是否准备好被读取（是否有可用字符）。
                    
                - **返回值**：`true` 表示可读取，`false` 表示可能阻塞。
                    
    2. **Writer详解：**
        
        1. **Writer概述：**Writer是 Java IO 中所有字符输出流的抽象父类 ，它为将字符数据写入各种字符数据目的地提供了基本的方法定义 。与字Reader相对应，Writer同样是抽象类，不能直接实例化，它的子类通过实现Writer的抽象方法，实现了针对不同字符数据的具体写入操作
            
        2. **Writer的常用方法：**
            
            1. **抽象方法（这些方法是 `Writer` 类中定义的抽象方法，所有子类必须实现它们。）**
                
            2. **`abstract void write(char[] cbuf, int off, int len) throws IOException`**
                
                - **作用**：将字符数组 `cbuf` 中从 `off` 索引开始的 `len` 个字符写入输出流。
                    
                - 参数：
                    
                    - `cbuf`：源字符数组。
                        
                    - `off`：起始索引。
                        
                    - `len`：要写入的字符数。
                        
            3. **`abstract void write(int c) throws IOException`**
                
                - **作用**：写入单个字符（`int` 类型的低 16 位会被转换为 `char`）。
                    
                - **参数**：`c` 是要写入的字符（0~65535）。
                    
            4. **非抽象方法（这些方法由 `Writer` 类提供默认实现，子类可以选择覆盖。）**
                
            5. **`void write(String str) throws IOException`**
                
                - **作用**：将字符串 `str` 的全部字符写入输出流。
                    
                - **参数**：`str` 是要写入的字符串。
                    
                - **等价于**：`write(str, 0, str.length())`。
                    
            6. **`void write(String str, int off, int len) throws IOException`**
                
                - **作用**：将字符串 `str` 中从 `off` 开始的 `len` 个字符写入输出流。
                    
                - **参数：**
                    
                    - `str`：源字符串。
                        
                    - `off`：起始索引。
                        
                    - `len`：要写入的字符数。
                        
            7. **`void write(char[] cbuf) throws IOException`**
                
                - **作用**：将字符数组 `cbuf` 的全部字符写入输出流。
                    
                - **参数**：`cbuf` 是源字符数组。
                    
                - **等价于**：`write(cbuf, 0, cbuf.length)`。
                    
            8. **`void flush() throws IOException`**
                
                - **作用**：刷新输出流，强制将缓冲区中的字符写入目标（如文件、网络等）。
                    
                - **注意**：某些流（如 `BufferedWriter`）内部有缓冲区，必须调用 `flush()` 才能立即写入数据。
                    
            9. **`void close() throws IOException`**
                
                - **作用**：关闭流并释放相关资源。
                    
                - **注意**：使用完流后必须调用 `close()`，否则可能导致资源泄漏。
                    
            10. **`Writer append(CharSequence csq) throws IOException`**
                
            
            - **作用**：将 `CharSequence`（如 `String`、`StringBuilder`）追加到输出流。
                
            - **参数**：`csq` 是要追加的字符序列。
                
            - **返回值**：`Writer` 实例本身，支持链式调用。
                
            
            11. **`Writer append(CharSequence csq, int start, int end) throws IOException`**
                
                - **作用**：将 `CharSequence` 中从 `start` 到 `end-1` 的字符追加到输出流。
                    
                - **参数：**
                    
                    - `csq`：源字符序列。
                        
                    - `start`：起始索引（包含）。
                        
                    - `end`：结束索引（不包含）。
                        
            12. **`Writer append(char c) throws IOException`**
                
                - **作用**：将单个字符追加到输出流。
                    
                - **参数**：`c` 是要追加的字符。
                    
                - **返回值**：`Writer` 实例本身，支持链式调用。
                    

## （三）按照功能进行分类

Java按照功能将流分为了两种：节点流与处理流。节点流也称为低级流，节点流的另一端是明确的，是实际读写数据的流，高级流一定建立在节点流的基础上进行的。处理流也称为高级流，处理流不能独立存在，必须链接在其他流上，目的是当数据流经过当前流时对数据进行加工处理来简化我们对数据的该操作。实际应用中，我们可以通过串联一组高级流到某个低级流上以流水线式的加工处理对某设备的数据进行读写，这个过程也称为流的链接，这也是IO的精髓

### 1.低级流（节点流）

低级流的特点：直接连接程序与另一端的管道，是真实读写数据的流。IO一定是建立在姐弟啊牛的基础上进行的。文件流就是典型的节点流

### 2.高级流（处理流）

高级流的特点：不能独立存在，必选链接在其他流上（不一定要链接在低级流上也可以链接在高级流上）

高级流的作用：当数据经过高级流时可以对数据进行某种加工操作来简化我们的同等操作。在实际开发中，我们经常串联一组高级流最终到某个低级流上，使读写数据以流水线式的加工处理完成，这一操作也常为流的链接，流的链接也是Java IO的精髓所在

## （四）按照数据操作对象进行分类

### 1.文件流

1. **文件流**：用来链接我们的程序与文件之间的“管道”，可以读写文件，文件流是一对低级流，用于读写文件的流。文件字节流分为
    
    1. 文件输入流，java.io.FileInputStream：读取文件数据。继承自InputStream
        
    2. 文件输出流：java.io.FileOutputStream：写入文件数据继承自OutputStream
        
    3. 文件流是一对低级流
        
2. **文件输入流：**java.io.FileInputStream使用文件输入流向从文件中读取数据
    
    1. 文件输入流构造器：
        
        1. FileInputStream(String path)基于给定的路径对应的文件，创建文件输入流。指定文件不存在则会抛出异常
            
        2. FileInputStream(File file)基于给定的File对象所表示的文件创建文件输入流。file对象表示的文件不存在则会抛出异常
            
        3. // 通过文件路径创建  
            FileInputStream fis = new FileInputStream("data.bin");  
              
            // 通过File对象创建  
            File file = new File("data.bin");  
            FileInputStream fis = new FileInputStream(file);  
              
            // 通过文件描述符创建（高级用法）  
            FileDescriptor fd = fis.getFD();  
            FileInputStream fis2 = new FileInputStream(fd);
            
    2. 文件输入流方法：参考其父类
        
3. 文件输出流：java.io.FileOutputStream使用文件输出流向文件中写入数据
    
    1. 文件输出流构造器：
        
        1. FileOutputStream(String path)，创建文件输出流对指定的path路径表示的文件进行写操作，如果该文件不存在则将其创建出来
            
        2. FileOutputStream(File file)创建文件输出流对指定的file对象表示的文件进行写操作，如果该文件不存在则将其创建出来
            
        3. （这两个构造方法都有一个重载的构造方法，在参数列表上加了一个布尔值，如果为true则是追加模式，false则是覆盖模式，一个参的构造方法都是默认false）
            
    2. 文件输出流方法：参考其父类
        
4. 文件复制：复制文件的原理就是使用文件输入流从原文件中陆续读取出每一个字节，然后再使用文件输出流将字节陆续写入到另一个文件中完成的。
    
5. 文本数据的写入与读取：
    
    1. **写出文本数据：**先将字符转换成二进制，通过String类的getBytes（）方法，返回一个byte数组。
        
    2. **读取文本数据：**先读取文件中的字节，然后将字节通过String构造器（通过new调用，传入一个字节类型的数组和编码格式）转换成字符串
        

### 2.对象流

1. 对象流：java.io.ObjectInputStream和ObjectOutputStream。对象输入流是将我们的Java对象进行序列化，对象输入流是将我们的Java对象进行反序列化。
    
    1. java.io.ObjectInputStream对象输入流，继承自java.io.InputStream
        
    2. java.io.ObjectOutputStream对象输出流，继承自java.io.OutputStream
        
2. 序列化：
    
    1. 序列化指的是将一个对象转换为一组可被传输或保存的字节。这组字节中除了包含对象本身的数据外，还会包含结构信息。
        
    2. 序列化的意义：在实际开发中，我们通常将对象写入磁盘进行长久保存和在两台计算机中的Java间通过网络进行传输。无论是保存在磁盘还是在传输都需要将对象转换为字节后才可以进行操作
        
    3. 序列化对象时要求该对象对应的类必须实现接口:java.io.Serializable，如果写出的对象对应的类没有实现该接口，那么writeObject方法会抛出下面异常java.io.NotSerializableException。
        
    4. 序列化接口java.io.Serrializable该接口没有任何抽象方法，但是只有实现了该接口的类的实例才能进行序列化与反序列化。
        
3. 对象输出流java.io.ObjectOutputStream使用对象流可以进行对象序列化
    
    1. 构造器：
        
        ObjectInputStream(InputStream in)  
        创建一个对象输入流并连接到参数in这个输入流上。
        
    2. 方法：
        
        void writeObject(Object obj)  
        将给定的对象转换为一组可保存或传输的字节然后通过其链接的流将字节写出
        
4. 对象输入流java.io.ObjectInputStream使用对象流可以进行对象反序列化
    
    1. 构造器：
        
        ObjectInputStream(InputStream in)  
        //将当前创建的对象输入流链接在指定的输入流上
        
    2. 方法：
        
        Object readObject()  
        //进行对象反序列化并返回。该方法会从当前对象输入流链接的流中读取若干字节并将其还原为对象。这里要注意读取的字节必须是由ObjectOutputStream序列化一个对象所得到的字节。
        

异常

## （一）异常处理机制

1. **Java的异常处理机制**：Java 的异常处理机制提供了一种结构化的方式来处理程序运行时可能出现的错误情况。当有程序出现问题时它能够将代码控制权从出错点转移到最近的异常处理器，以一种可控的方式进行处理，而不是突然崩溃。通过对异常的处理，程序员可以更好地预测和处理可能出现的问题，提高程序的稳定性和可靠性。
    
    > 开发过程中的语法错误和逻辑错误不是异常，发生异常时java会阻止当前方法或作用域的情况。
    
2. **异常**：异常就是程序在运行过程中出现的一些错误，我们通过面向对象的思想，把这些错误也用类来描述，那么一旦产生一个错误，即就是创建了某一个错误的对象，这个对象就是我们所说的异常对象。
    

## （二）异常体系结构

1. **Java异常体系结构：** _java.lang.Throwable_，_Throwable_类是 Java 语言中所有错误或异常的超类，其下有两个异常分支，分别是 _java.lang.Error_ 和 _java.lang.Exception_，代码中抛出的异常通常指 _java.lang.Exception_。![img](https://c.biancheng.net/uploads/allimg/250811/2-250Q1102100630.gif)![Java异常的体系结构](https://img-blog.csdn.net/20160331115514210)
    
    1. Error 由Java虚拟机抛出，是程序中无法处理的错误，表示运行应用程序中出现了严重的错误，。此类错误一般表示代码运行时JVM也就是内部系统出现问题。当此类错误发生时，应用不应该去处理此类错误，而除了通知用户以及尽量稳妥地终止程序外，几乎什么也不能做。
        
    2. Exception在编译时期抛出的异常，在编译期间检查程序是否可能会出现问题，如果可能会有，则预先防范：捕获 声明。异常产生后，程序员可以通过代码方式纠正，使程序继续运行。Exception 又可以细分为运行时异常和非运行时异常，也称为未检查异常和检查异常。
        
        - **非运行时异常(受检异常)**： Exception中除RuntimeException极其子类之外的异常。编译器会检查此类异常，如果程序中出现此类异常，编译器会强制程序员检査并通过 try- catch 块处理它们，或者在方法头进行声明。
            
        - **运行时异常（未检异常）**都是 RuntimeException 类及其子类，这些异常是未检查异常。在程序中，你可以选择捕获和处理这些异常，也可以不理会。这些异常通常是由程序中的逻辑错误引起的，程序应该尽可能避免此类异常。RuntimeException 类属于非检测异常，因为普通JJVM操作引起的运行时异常随时可能发生，此类异常一般是由特定操作引发。但这些操作在j在java应用程序中会频繁出现。因此它们不受编译器检查与处理或声明规则的限制。实际上RuntimeException及其子类型表达的都是因为程序漏洞(BUG),即:逻辑不严谨等原因导致的。这类异常都是通过修复代码可完全避免的异常，因此不应当由异常处理机制来处理。 RuntimeException 的任何子类都无需在 throws 子句中进行声明，指的就是这些问题不需要提前被预防（本质上也可以的，只不过没必要），因为只有在真正运行的时候才能发现是否发生问题，一旦在运行期间发生了问题，则一般不会修正，程序直接中断。
            
        - **从程序执行的过程来看编译时异常和运行时异常**
            
            - 编译时异常：程序在编译时发生的异常（javac 源文件名.java）
                
            - 运行时异常: 程序在运行时发生的异常（java 字节码文件名）
                
    3. Java规定：必须捕获的异常，包括 Exception 及其子类，但不包括 RuntimeException 及其子类，这种类型的异常称为Checked Exception。 不需要捕获的异常，包括 Error 及其子类， RuntimeException 及其子类。
        

## （三）异常的处理

1. **异常的抓抛模型原理**：
    
    - **异常的抛出**：如果程序在运行过程中执行某段代码时发生了异常，那么系统（JVM）将会根据异常的类型，在异常代码处创建对应的异常类型对象并抛出，抛出给程序的调用者。一旦抛出对象以后,其后的代码不再运行,程序终止.异常的抛出分为：① 系统向外抛出异常 ② 手动向外抛出异常（throw）
        
    - **异常的抓取**：异常的抓取可以理解为异常的处理方式,也就是抓住抛出的异常。 取有 try-catch-finally 和 throws 两种方式
        

### 1.声明异常（throws关键字）

1. `throws` 是 Java 异常处理机制中的核心关键字，用于**声明方法可能抛出的受检异常（checked exceptions）**（可以抛出非受检异常，但编译器不强制要求）。每个方法都必须声明它可能抛出的受检异常的类型，这称为声明异常（ declaring exception)。告知方法调用者该方法存在异常，必须处理异常，否则编译不通过。处理手段有两种:使用try-catch捕获并处理这个异常，或者继续throws声明。
    
    > 注意，永远不应当在main方法上使用throws!!
    
2. 基本语法：方法签名处声明异常类型，多个异常用逗号隔开。
    
    返回类型 方法名(参数列表) throws 异常类型1, 异常类型2, ... {  
        // 方法体  
    }
    
    > throws的方式只是将异常抛给了方法的调用者，并没有真正将异常处理掉。
    
3. **throws的重写规则：**（只能小不能大）当一个子类重写超类含有throws声明异常抛出的方法时，针对throws的重写规则：
    
    1. 子类重写的方法抛出的异常类型不大于父类被重写的方法抛出的异常类型（数量，子类）
        
    2. 如果基类中被重写的方法没有throws方式处理异常,则子类重写的方法也不能使用throws,意味着如果子类重写的方法中有异常,必须使用tyr-catch-finally方式处理.
        
        > 注意: try-catch和throws在方法中不要同时使用,因为只要使用try-catch就已经将异常处理掉了,再throws没有任何意义.
        

### 2.抛出异常（throw关键字）

1. **抛出异常：**`throw` 是 Java 异常处理机制的核心关键字，用于**在代码中显式抛出异常对象**。与 `throws`（声明异常）不同，`throw` 是实际执行异常抛出的动作。
    
2. throw是我们手动抛出的异常，
    
3. 当我们察觉程序可能出现的错误时，可以手动创建一个合适的异常类型实例并抛出它，这就称为抛出一个异常（throwing an exception)。
    
4. 一旦方法体执行时,出现异常，会在异常代码处生成一个异常类的对象，此对象满足抛出的异常类型时，就会被抛出。异常代码的后续的代码，就不再执行！
    
5. throw用来对外主动抛出一个异常，通常下面两种情况我们主动对外抛出异常:当程序遇到一个满足语法，但是不满足业务要求时，可以抛出一个异常告知调用者。程序执行遇到一个异常，但是该异常不应当在当前代码片段被解决时可以抛出给调用者。手动抛出的异常有两种, 分别为**运行时异常**和**编译时异常**.抛出运行时异常时, 可以不用处理抛出的这个异常.抛出编译时异常时, 必须要处理抛出的这个异常.  
    
    ExceptionName ex = new ExceptionName("Wrong");  
    throw ex;  
      
    或者  
        
    throw new ExceptionName("Wrong");
    
    > 通常，JavaAPI 中的每个异常类至少有两个构造方法：一个无参构造方法和一个带可描述这个异常的 String 参数的构造方法。该参数称为异常消息（exceptionmessage), 它可以用 getMessage()获取。
    
6. JVM----->调用main----->调用其他方法----->其他方法出现异常----->抛出给main----->main没有处理----->抛出传递给JVM----->JVM直接中断。（函数内部如果出现问题，也可以将问题在内部处理 处理后 就不需要向上层抛出。）
    

### 3.捕获异常（异常的处理）（try-catch-finally关键字）

1. **捕获异常（异常的处理）：**当抛出一个异常时，可以在try-catch 块中捕获和处理它；
    
2. **try - catch - finally**
    
    异常处理机制中的try-catch - finally  
    	语法:  
    	try{  
        代码片段...  
    	}catch(XXXException e){  
        出现错误后的补救措施(B计划)  
        //这里实际开发中是写补救措施的，通常也会将异常信息输出便于debug  
    	}finally{  
        //代码  
      }
    
3. **try**：
    
    1. 将可能出现异常的代码放到try{}中, 运行时, 如果代码发成了异常的话,就会生成一个对应的异常类的对象.这个产生的异常对象与catch的异常类型相匹配就会被 catch 捕获,然后运行catch{}中的代码, 一般catch中的代码为处理异常的代码。一旦异常处理完成,就会跳出当前的try-catch结构，因此try语句块中报错代码以下的内容都不会被执行
        
    2. try{}语句块不能单独写，必须至少跟随一个 catch 或 finally 块。
        
    3. 块内变量作用域仅限于 try 块内部（大括号为一个作用域）
        
4. **catch**：
    
    1. catch用来捕获并处理特定类型的异常，可以定义多个catch，也可以合并多个异常在一个catch中（捕获并处理特定类型的异常，用`|`隔开）。
        
    2. 当catch捕获某个超类型异常时，那么try语句块中出现它类型的子类异常时都可以被这个catch块捕获并处理。
        
5. **Finally：**
    
    1. finally块是异常处理机制中的最后一块，finally可以直接跟在try语句块之后，finally也可以跟在最后一个catch块之后，fianlly下面不能再定义catch块。
        
    2. finally块总是执行，无论try中的代码是否出现异常。
        
    
    > try-catch-finally结构可以相互嵌套
    
6. **自动关闭特性（try-with-resources）**：JDK7新特性，自动关闭实现 `AutoCloseable` 的资源。关闭顺序与声明顺序相反。
    

### 4.异常API

Exception上有一些常用的方法：

- void printStackTrace():用于输出当前异常的堆栈跟踪信息，便于程序员找到错误根源，分析错误原因制定B计划
    
- String getMessage():用于获取当前错误消息，多用于提示给用户或记录日志使用
