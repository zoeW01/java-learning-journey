### 一、变量是什么

变量 = 一块被命名的内存空间。声明变量时JVM做三件事：
1. 根据类型分配内存空间（如int分配4字节，byte分配1字节）
2. 把初始值的二进制形式存入这块空间
3. 把这块空间命名为变量名

### 二、八种基本数据类型

| 数据类型  | 关键字       | 占用字节数 | 取值范围            | 示例                        |
| ----- | --------- | ----- | --------------- | ------------------------- |
| 字节型   | `byte`    | 1     | -128 ~ 127      | `byte b = 100;`           |
| 短整型   | `short`   | 2     | -32768 ~ 32767  | `short s = 30000;`        |
| 整型    | `int`     | 4     | 约 ±21 亿         | `int i = 2000000000;`     |
| 长整型   | `long`    | 8     | 极大              | `long l = 900000000000L;` |
| 单精度浮点 | `float`   | 4     | 小数              | `float f = 3.14f;`        |
| 双精度浮点 | `double`  | 8     | 更精确小数           | `double d = 3.1415926;`   |
| 布尔型   | `boolean` | 1     | true/false      | `boolean b = false;`      |
| 字符型   | `char`    | 2     | 单个字符（0 ~ 65535） | `char c = '中';`           |

**记忆要点：**
-  整数默认是 `int` 类型，小数默认是 `double` 类型
    
- `long` 类型的值后面必须加 `L`，否则编译器会当成 `int`
    
- `float` 类型的值后面必须加 `f`，否则编译器会当成 `double`

### 三、类型转换

**自动类型转换（隐式）**：小范围赋值给大范围，自动安全
```java
int a = 100;
long b = a;     // int → long，自动
double c = a;   // int → double，自动
```

**强制类型转换（显式）**：大范围赋值给小范围，必须手动写`(目标类型)变量名`，可能丢数据
```java
double d = 9.99;
int i = (int) d;    // i = 9，小数截断，不是四舍五入
```

基本数据类型大小：byte、short、char → int → long → float → double。  


**表达式中的自动类型提升**

**规则**：byte、short、char参与算术运算时，先自动提升为int再计算，结果也是int。

**原因**：JVM没有专门为小于int的类型设计算术运算指令，底层硬件寄存器也是32位起步。

```java
byte a = 1;
byte b = 2;
byte c = a + b;          // 报错。a+b结果是int，int不能自动转byte
byte c = (byte)(a + b);  // 正确。强制转换
int c = a + b;           // 正确。用int接
```

**提升链**：byte/short/char → int → long → float → double

### 四、两个特殊编译规则

**规则1：编译时常量优化**
```java
byte b = 25;     // 通过。25是常量，且在byte范围内，编译器自动处理
byte b = 200;    // 报错。200超出范围
```
