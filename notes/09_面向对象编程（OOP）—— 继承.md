
- 继承是面向对象最强大的机制之一。它让一个类可以继承另一个类的属性和方法，避免重复代码。

- **现实类比：** "学生"和"老师"都是"人"。人有姓名和年龄，有介绍自己的方法。学生有成绩，老师有科目。与其在两个类里重复写姓名和年龄的代码，不如让它们都继承自"人"。

- **在 OOP 里：**

	- `Person` 是父类（基类）
	    
	- `Student` 和 `Teacher` 是子类（派生类）
	    
	- 子类自动拥有父类的属性和方法，还可以添加自己特有的东西


## Java 继承特点
- **单继承**：一个类只能有一个直接父类。
- 可以多重继承（A extends B extends C），呈链条状。
- `Object` 是所有类的最终祖先，没有写 `extends` 的类默认继承 `Object`。

## 关键语法 `extends` 和 `super`：

- ```java
	class 子类 extends 父类 {
	    // 子类自动拥有父类的非私有成员
	}
```
    
- `super()` 调用父类构造方法。子类构造器**第一行必须调用父类构造器**：
    
- 子类 不需要再写 父类拥有的属性和方法——它们可以从 父类 继承而来（子类自动拥有父类的所有 public 方法和属性）
    
- 编译器会自动插入 `super()`，调用父类无参构造器。父类没有无参构造器时，必须手动写 `super(参数)`，否则编译报错


# 方法重写

**方法重写的规则：**

- 在父子类中，方法名与参数列表相同。
- 返回类型相同或为其子类型。  
- 访问权限不能比父类更严格。 
- 抛出的受检异常不能比父类更多。
- 不能重写`private`、`static`、`final`方法。  



# `super` 关键字

**三大用途**：
1. `super.变量名`：调用父类被隐藏的成员变量
2. `super.方法名(参数)` ：调用父类被重写的方法
3. `super()` ：调用父类构造方法，必须在子类构造方法第一条。

**致命规则**：
- `super(参数)` 必须在子类构造方法体的**第一条语句**。
- 如果子类构造方法没写 `super(...)`，编译器自动插入 `super()` 调用父类无参构造。
- **因此如果父类没有无参构造方法，子类构造方法必须显式写 `super(参数)`，否则编译错误。**
