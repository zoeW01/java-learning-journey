
### 1. Java的诞生、发展历程

**必记数据**：
- Java前身叫 **Oak**，1995年正式发布（简称Java 1.0）。
- Sun公司开发，**James Gosling**（詹姆斯·高斯林）是核心创建人。
- 2010年Oracle收购Sun，Java归属Oracle。


### 2. Java语言的特点

| 特点   | 关键解释                       |
| ---- | -------------------------- |
| 简单性  | 去掉了C++的指针、多继承、运算符重载        |
| 面向对象 | 一切皆为对象，符合OOP三大特性（封装、继承、多态） |
| 分布式  | 支持网络编程，提供URL、Socket类库      |
| 健壮性  | 强类型、自动垃圾回收、异常处理            |
| 安全性  | 无指针、字节码校验、Applet沙箱机制       |
| 平台无关 | 一次编译，到处运行，靠**JVM实现**       |
| 多线程  | 语言级内置多线程支持                 |

**致命混淆点**：
- 把"解释型"当成Java特点——这是错的。Java是**先编译成字节码，再解释执行**。
- 简答题漏掉任何一个特点都扣分，必须写全7个。

**真题变式**：选择题会问你"Java实现跨平台的核心是什么"，答案必须选**JVM**，不是编译器、不是JDK、不是字节码。

### 3. Java程序类型

**两个核心区别**：

|      | Application | Applet                     |
| ---- | ----------- | -------------------------- |
| 运行位置 | 本地JVM       | 浏览器                        |
| 入口方法 | 必须有`main()` | 无`main()`，用`init()`等生命周期方法 |
| 安全限制 | 无           | 沙箱限制，不能读写本地文件              |

**Java程序分为哪两种类型？简述区别。**

Application（应用程序）：独立运行在本地JVM上，必须有`main()`方法作为入口。  
Applet（小程序）：嵌入HTML页面，由浏览器中的JVM执行，无`main()`方法，由生命周期方法驱动，受沙箱安全限制

**易错点**：Applet不是独立的，必须嵌入HTML页面运行。选择题问你"以下哪类程序不需要main方法"，答案选Applet。

### 4. Java程序的开发流程

**三步流程**：
1. **编辑** → 使用文本编辑器编写源程序,产生 `.java` 源文件
2. **编译** →使用`javac`命令将`.java`文件编译为字节码`.class`字节码文件
3. **运行** →使用`java`命令启动JVM，解释执行字节码。

**高频陷阱**：
- 问你"Java编译器输出的文件扩展名是什么"，答案是`.class`，不是`.java`。
- 问你"java命令运行程序时，类名要不要加.class后缀"，答案是**不加**。例如`java HelloWorld`，不是`java HelloWorld.class`。

### 5. JDK常用工具

**考试定位**：选择题或填空题，2-3分。

**必须区分清楚四个工具**：

| 工具        | 功能      | 记忆技巧                |
| --------- | ------- | ------------------- |
| `javac`   | 编译器     | c = compile         |
| `java`    | 解释器/启动器 | 运行                  |
| `javadoc` | 文档生成器   | doc = documentation |
| `jar`     | 打包工具    | 把.class打包成jar       |
| `jdb`     | 调试器     | db = debug          |

**易错真题**："生成API文档需要用什么命令"，答案是`javadoc`。很多人写成`javadoc`后面漏参数直接扣分，但选择题问到工具名称，选`javadoc`。

### 6. 源文件命名规则

**三条铁律**：
1. 如果有`public`类，**文件名必须与该public类名完全一致**（大小写敏感）。
2. 一个`.java`文件最多只能有一个`public`类。
3. 如果没有`public`类，文件名可以任意命名。

-  Java 源文件的基本编译单位是类

**经典陷阱题**：
- "一个Java源文件中可以包含多个类吗？"——**可以**，但public类最多一个。
- 问你编译后会产生几个`.class`文件？有几个类就产生几个，和文件数量无关。

### 7. main方法的格式及各部分含义

格式：`public static void main(String[] args)`  
- `public`：JVM需要从外部调用，必须公开。  
- `static`：JVM调用时无需创建对象。  
- `void`：main方法不返回任何值给JVM。  
- `main`：JVM识别的固定入口方法名。  
- `String[] args`：接收命令行参数，参数名可自定义。

---

### 8. 标识符命名规则

**四条规定**：
1. 只能包含**字母、数字、美元符`$`、下划线`_`。**
2. 数字不能开头。
3. 不能用**关键字**（如`class`、`int`、`public`）。
4. 区分大小写，`name`和`Name`是两个标识符。

---

### 9. 注释

**三种注释**：

| 类型 | 格式 | 用途 |
|---|---|---|
| 单行 | `// 内容` | 一行 |
| 多行 | `/* 内容 */` | 多行 |
| 文档 | `/** 内容 */` | 生成javadoc |

**唯一坑点**：多行注释**不能嵌套**。`/* a /* b */ c */` 编译报错。选择题经常用这个出题，问"以下注释是否正确"。
