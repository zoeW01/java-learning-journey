
- 这是 Java 最核心的概念。之前的代码都是放在 `main` 方法里，属于面向过程——一步一步执行指令。面向对象是换一种组织代码的方式：把数据和操作数据的方法打包在一起，形成“对象”。

# 类与对象

- 现实世界的类比：一辆车是一个对象。
	- **属性（数据）**：颜色、品牌、速度
	
	- **行为（方法）**：加速、刹车、鸣笛


- 类就是造车的图纸，是**一类事物的共同模板**。对象是按照图纸造出来的具体的车。一张图纸可以造无数辆车——一个类可以创建无数个对象。

类定义的结构：
```java
[修饰符] class 类名 {
    // 构造方法
    // 成员变量（属性）
    // 成员方法（行为）
}
```
一个源文件可以定义多个类，但 `public` 修饰的类只能有一个，且必须与文件名一致。

---

### 属性：对象拥有的数据

属性声明在类内部、方法外部。每个对象都有自己的一份属性副本，互不影响。

属性默认值规则与数组元素一致：数值类型为0/0.0，boolean为false，引用类型为null。

---

### 方法：对象拥有的行为

方法属于对象，可以访问和操作对象的属性。

```java
[修饰符] 返回类型 方法名(参数列表) {
    方法体
    [return 值;]
}
```

- **返回值**：方法执行完后交还给调用者的结果。`void` 表示不返回任何东西。
- **参数**：调用者传入的数据，方法内部用参数名引用它。
- **return**：结束方法并返回结果。void方法里 `return;` 表示直接结束。

方法名可以相同、参数名可以相同——JVM通过**对象 + 方法名 + 参数类型**三者组合来区分调用。

- new关键字就是用来创建对象的，new会在堆上开辟空间，然后返回该地址的首地址。

- 创建对象后使用`.`来调用属性和方法

# 构造方法：对象创建时自动执行的特殊方法

**构造方法特点：**
① 方法名必须与类名完全一致。  
② **没有返回值类型**，连`void`都不能写。  
③ 每个类至少有一个构造方法，不写时编译器自动添加默认无参构造方法。  
④ 一旦自定义了任意构造方法，默认无参构造方法消失。  
⑤ 可以重载。
⑥`new`对象时自动调用，作用是初始化对象


- 关键点：一旦你自己写了一个构造器（不管带不带参数），Java就**不再自动提供**默认构造器。

# `this`关键字

- `this` 代表“当前这个对象”。
- 在方法调用时，调用者会作为参数传入到方法中的 `this` 变量中。

**三大用途**：
- `this.属性名;`  访问当前对象属性，区分成员变量和局部变量同名 
- `this();` 调用本类的构造方法，只能在构造方法内部使用且位于方法首行（`this()` 和 `super()` 在通一个构造方法中只能存在一个）
- `this.方法名()`;  调用当前对象的方法，区分父子类的方法

- `this` 可以作为方法参数使用，也可以作为方法的返回值返回

# 封装

- 封装就是把数据藏起来，只暴露安全的方法给外部使用。

- 步骤：
	1.  **将成员变量用 `private` 修饰**
	
	2. **提供 getter 和 setter 方法**。getter 用来读取，setter 用来修改。在 setter 里可以做数据校验。

---

# 方法重载（Overload）与方法重写（Override）

**方法重载**

- **重载条件**：在同一个类中或者父子类中，方法名相同但参数列表不同（个数、类型、顺序至少一项不同），即为重载。
- **返回类型**不能作为重载区分条件。
- 访问权限可以任意
- 重载是编译时多态、重写是运行时多态

**方法重写**

- 重写条件：在父子类中，方法名和参数列表相同即为重写。
- 返回类型相同或为其子类型。  
- 访问权限不能比父类更严格。 
- 父类方法不能是`private`/`static`/`final`
- 抛出的受检异常不能比父类更宽泛（子类方法声明的受检异常必须是父类方法所声明的受检异常或其子类（或者不抛））
- 重载是编译时多态、重写是运行时多态

# `static`关键字

- 目前为止所有方法和变量都属于“对象”。有的东西不依赖于任何具体对象，只属于类本身，就用 `static`。

- 调用时不用使用new来创建对象，直接用类名调用

**静态变量**：
- 属于类，不属于任何一个对象。
- 使用 `类名.静态变量` 访问（当然也可通过对象访问，但不推荐）。

**静态方法**：
- 只能访问静态成员（变量、方法），**不能直接访问实例成员**。
- 不能用 `this` 和 `super`。
- 可以通过类名调用，不需要创建对象（当然也可通过对象访问，但不推荐）。

**静态初始化块**：
```java
static {
    // 类加载时执行一次，用于初始化静态变量
}
```
静态初始化块在类加载时执行，且只一次。

## 访问控制修饰符

| 修饰符            | 同一类 | 同一包 | 子类  | 任何地方 |
| -------------- | --- | --- | --- | ---- |
| `private`      | ✓   |     |     |      |
| `default`（无修饰） | ✓   | ✓   |     |      |
| `protected`    | ✓   | ✓   | ✓   |      |
| `public`       | ✓   | ✓   | ✓   | ✓    |

# `final` 关键字

final：到此为止，不能修改

| 修饰       | 效果          |
| -------- | ----------- |
| final 类  | 不能被继承       |
| final 方法 | 不能被子类重写     |
| final 变量 | 只能赋值一次，变成常量 |
`final` 和 `abstract` 互斥——一个强制被重写，一个禁止被重写。

# 内部类

 内部类的分类：

① 成员内部类：定义在类中与方法平级。  
② 局部内部类：定义在方法体内部。  
③ 匿名内部类：没有名字，常用于接口/抽象类的实例化。  
④ 静态内部类：用`static`修饰，只能访问外部类的静态成员。


**成员内部类访问特点**：
```java
class Outer {
    private int x = 10;
    class Inner {
        void show() {
            System.out.println(x); // 可以访问外部类的私有成员
        }
    }
}
```
创建内部类对象需要先有外部类对象：`Outer.Inner i = new Outer().new Inner();`

**匿名内部类**（重点）：
```java
Thread t = new Thread() {
    public void run() {
        System.out.println("running");
    }
};
```
本质是定义一个没有名字的类继承 Thread 并覆写 run 方法，同时创建该子类对象。

**局部内部类**：
- 定义在方法内，作用域仅在方法内。
- 可以访问外部类的成员，以及方法中的 `final` 局部变量（Java 8 之后局部变量若未发生改变也被视为事实 final）。
