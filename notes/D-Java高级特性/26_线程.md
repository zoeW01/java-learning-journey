简述创建线程的两种方式及优缺点。**

方式一：继承`Thread`类，重写`run()`。优点：代码简单。缺点：单继承限制，不能再继承其他类。  
方式二：实现`Runnable`接口，重写`run()`，传给`Thread`构造器。优点：可同时继承其他类，适合资源共享。缺点：代码稍复杂。



# 线程

## （一）单线程

1. 线程概念：一个顺序的单一的执行流程就是一个线程。代码一聚一聚先后顺序的执行。
    
2. 顺序执行：前面的干不完，后面的不能干，也称为串联方式
    
3. 如果想让计算机一下子可以干多件事情怎么办
    

## （二）多线程

1. 多线程概念：多个单一顺序执行的流程并发运行，利用高性能造成感官上同时运行的效果。也就是多个线性流程一起执行，多线程的本质就是一次干多件事情
    
2. 并发概念：微观上是走走停停的，宏观上是同时运行。操作系统的线程调度器（CPU大管家）尽可能均匀的分配时间片段，得到时间片段的线程被CPU运行（可能连续分配数次时间片段给同一个线程），其他线程全部等待，CPU在利用高性能模仿同时。
    
    > CPU来运行代码，CPU一次只能执行一条指令，一次只能干一件事情。CPU发展，单核心到多核心。晶体管和振荡频率。多核并发高效率
    
3. 并发执行：并联方式，其实还是一次干一件事情，但是中途会干其他事情
    
4. 多线程的使用场景：
    
    1. 同时处理多个互不打扰的任务
        
    2. 单线程可以完成，多线程可以更快
        
5. 线程的创建
    
    1. 方式一：
        
        1. 创建一个继承于Thread类的子类。
            
        2. 重写Thread类的run()方法。
            
        3. 创建Thread类的子类的对象。
            
        4. 通过此对象调用start()来启动一个线程。
            
        5. 优点：结构简单，利于匿名内部类创建
            
        6. 缺点：
            
            1. 存在继承冲突问题，由于java是单继承的，这导致如果我们继承了Thread就无法再继承其他类去复用方法。这在实际开发中是极其不方便的
                
            2. 当我们继承Thread并重写run方法，在run方法中定义线程要执行的任务。这会导致线程与任务存在一个必然的耦合关系，不利于线程的重用。
                
        
        public class ThreadDemo01 {  
            public static void main(String[] args) {  
                Thread t1 = new Thread01();  
                Thread t2 = new Thread02();  
                t1.start();  
                t2.start();  
            }  
        }  
        class Thread01 extends Thread{  
            public void run(){  
                int i=0;  
                while (true){  
                    i++;  
                    long end = System.currentTimeMillis();  
                    if (i%1000000000L==0){  
                        System.out.println("我是线程01");  
                    }  
                }  
            }  
        }  
          
        class Thread02 extends Thread{  
            public void run(){  
                int i=0;  
                while (true){  
                    i++;  
                    long end = System.currentTimeMillis();  
                    if (i%1000000000L==0){  
                        System.out.println("asdasufgaghfu");  
                    }  
                }  
            }  
        }
        
    2. 方式二：实现Runnable接口
        
        1. 创建一个实现Runnable接口的类。
            
        2. 实现类去实现Runnable接口中的抽象方法：run()。
            
        3. 创建实现类的对象。
            
        4. 将此对象作为参数传到Thread类的构造器中，创建Thread类的对象。
            
        5. 通过Thread类的对象调用start()方法。
            
        6. 优点：
            
        
        - 没有继承冲突
            
        - 将任务和线程分离，不存在耦合，可以进行线程的重复利用
            
        
        public class ThreadDemo02 {  
            public static void main(String[] args) {  
                Runnable r1= new MyRunnable01();  
                Runnable r2= new MyRunnable02();  
          
                Thread t1=new Thread(r1);  
                Thread t2=new Thread(r2);  
          
                t1.start();  
                t2.start();  
            }  
        }  
          
        class MyRunnable01 implements Runnable{  
            public void run(){  
                for (int i = 0; i < 1000; i++) {  
                    System.out.println("我是线程01");  
                }  
            }  
        }  
          
        class MyRunnable02 implements  Runnable{  
            @Override  
            public void run() {  
                for (int i = 0; i < 1000; i++) {  
                    System.out.println("按时交付给i哦发表");  
                }  
            }  
        }
        
    3. 方式三：基于第二种方法使用匿名内部类。Runnable接口可以使用Lambda表达式的方式创建。相当于给Thread的构造函数传了一个Runnable的匿名对象，然后又给这个匿名对象创建了一个匿名内部类，最后再用Lambda表达式直接替代完整写法。因为`Runnable`是函数式接口（只有一个抽象方法`run()`），编译器会自动将Lambda表达式`() -> { ... }`解释为`Runnable`的匿名实现，无需显式写`new Runnable`或方法名。
        
        Thread t = new Thread(()->{  
            for (int i = 0; i < 1000; i++) {  
                System.out.println("我是线程一号！");  
            }  
        });  
          
        等价于以下代码：  
             
        Thread t1 = new Thread(new Runnable() {  
            @Override  
            public void run() {  // 实现Runnable的唯一抽象方法  
                for (int i = 0; i < 1000; i++) {  
                    System.out.println("我是线程一号！");  
                }  
            }  
        });
        
6. 主线程：Java中的代码都是靠线程执行的，main方法也不例外，执行main方法的线程称为主线程。Thread提供了一个静态方法，static Thread currentThread（），该方法可以将执行该方法的线程返回。线程也重写了toString方法打印出来是：Thread[线程名字，线程优先级，线程组]
    
7. 线程的生命周期
    
    1. NEW，新建：当一个Thread类或其子类的对象被声明并创建时，新的线程对象处于新建状态。
        
    2. RUNNABLE，就绪：处于新建状态的线程被start()后，将进入线程队列等待CPU时间片，此时它已具备了运行的条件，只是没分配到CPU资源。
        
    3. RUNNING，运行：当就绪的线程被调度并获得CPU资源时，便进入运行状态，run()方法定义了线程的操作和功能。
        
    4. SLEEP,WAIT,IO,阻塞：在某种特殊情况下，被认为挂起或执行输入输出操作时，让出CPU并临时中止自己的执行，进入阻塞状态。阻塞时，CPU强制释放其时间片段
        
    5. DEAD，死亡：线程完成了它的全部工作或线程被提前强制性的中止或出现异常倒置导致结束。
        

## （三）Thread类常用方法

1. **public void start()** : 启动当前线程, 调用当前线程的run()方法
    
2. **public void run()** : 通常需要重写Thread类中的此方法, 将创建的线程要执行的操作声明在此方法中
    
3. getName() : 获取当前线程的名字
    
4. **public final void setName(String name)**: 设置当前线程的名字
    
5. **public final void setPriority(int priority)**:更改线程的优先级。
    
6. **public final void setDaemon(boolean on)**“”将该线程标记为守护线程或用户线程。
    
7. **public final void join(long millisec)**: 在线程a中调用线程b的join(), 此时线程a进入阻塞状态, 直到线程b完全执行完以后, 线程a才结束阻塞状态
    
8. stop() : 已过时. 当执行此方法时,强制结束当前线程.
    
9. **public void interrupt()**：中断线程
    
10. **public final boolean isAlive()** ：测试线程是否处于活动状态。
    
11. **public static void yield()**：线程提供的这个静态方法作用是让执行该方法的线程主动放弃本次时间片。
    
12. **public static void sleep(long millisec)**：运行该方法的线程进入阻塞状态指定的毫秒,超时后线程会自动回到RUNNABLE状态等待再次获取时间片并发运行。该方法会声明抛出一个异常，调用方法需要捕获这个异常。使用线程的interrupt（）方法可以中断一条政治sleep阻塞的线程，此时sleep方法会立即抛出中断异常InterruptException。
    
13. **public static boolean holdsLock(Object x)**：当且仅当当前线程在指定的对象上保持监视器锁时，才返回 true。
    
14. **public static Thread currentThread()**：返回对当前线程
    
15. **public static void dumpStack()**：将当前线程的堆栈跟踪打印至标准错误流。
    

## （四）多线程并发安全（synchronized关键字）

**多线程并发安全问题概念：**当多个线程并发操作同一临界资源,由于线程切换时机不确定,导致操作临界资源的顺序出现混乱严重时可能导致系统瘫痪.。临界资源:操作该资源的全过程同时只能被单个线程完成.常见的临界资源：多线程共享实例变量，多线程共享静态公共变量

synchronized是Java中的一个关键字，用于实现**线程同步**，确保线程安全。synchronized可以修饰方法或者代码块。

1.同步方法：使用synchronized修饰的方法即为同步方法。对于实例方法，锁是当前对象（`this`）；对于静态方法，锁是类对象（`Class`）。

2.同步块：使用synchronized修饰的语句块即为同步块。同步块允许指定锁对象和同步范围以此来提供更精细的控制。

3.同步监视器对象：同步监视器对象即上锁的对象，同步监视器对象必须是引用类型且多个线程必须看到同一个同步监视器对象才能实现同步

1. 异步操作：多线程并发的操作，相当于各干各的
    
2. 同步操作：有先后顺序的的操作，相当于你先干我再干
    

Stringbuilder非并发安全，StringBuffer并发安全

集合都是非并发安全，可以使用Collections.synchronizedList（）方法将集合转为并发安全的集合

## 线程池

1. 是线程的管理机制，可以复用线程并且控制线程数量。线程池API
    
    创建任务 Runnable，调用pool.execute（任务）
    
    线程池关闭：shutdown（），不再接收新任务，将现有任务执行完毕，结束池中所有线程，shutdownNow（），不再接收新任务，中断线程池所有任务（sleep线程被中断会报错）



章 输入/输出流

### 考点1：流的概念与分类

**考试定位**：选择题给一个类名判断属于哪种流。

**流的本质**：程序与数据源/目的地之间的数据传输通道。

**按数据单位分**：

| 类型 | 抽象基类 | 输入 | 输出 |
|------|---------|------|------|
| 字节流 | 以字节（8位）为单位 | `InputStream` | `OutputStream` |
| 字符流 | 以字符（16位）为单位 | `Reader` | `Writer` |

**按流向分**：输入流（读入程序）、输出流（从程序写出）。


Java I/O 体系里，这四个都是**抽象类**，不是接口：

|抽象类|类型|方向|
|---|---|---|
|`InputStream`|字节输入流|读|
|`OutputStream`|字节输出流|写|
|`Reader`|字符输入流|读|
|`Writer`|字符输出流|写|

判断方法：看名字里有没有 `Stream`。有 `Stream` 的是字节流，有 `Reader`/`Writer` 的是字符流。它们全都是抽象类。



**易混淆**：
- `InputStream` 和 `OutputStream` 是**抽象类**，不能直接 `new`。
- 字节流处理二进制数据（图片、视频、所有文件），字符流专门处理文本。
- 字符流 = 字节流 + 编码表。本质是对字节流的包装。

### 考点2：文件字节流

**考试定位**：选择题判断构造方法参数，填空题补全代码。

**`FileInputStream`**（读文件）：
```java
FileInputStream fis = new FileInputStream("test.txt");
// 或
FileInputStream fis = new FileInputStream(new File("test.txt"));
```

**`FileOutputStream`**（写文件）：
```java
FileOutputStream fos = new FileOutputStream("test.txt");
// 或追加模式
FileOutputStream fos = new FileOutputStream("test.txt", true);
```

**关键方法**：

| 方法                     | 作用                         |
| ---------------------- | -------------------------- |
| `int read()`           | 读取一个字节，返回0~255，文件末尾返回 `-1` |
| `int read(byte[] b)`   | 读取多个字节，返回实际读取字节数，末尾返回 `-1` |
| `void write(int b)`    | 写出一个字节                     |
| `void write(byte[] b)` | 写出整个字节数组                   |
| `void close()`         | 关闭流                        |

**经典读程序模板**：
```java
int data;
while ((data = fis.read()) != -1) {
    System.out.print((char) data);
}
fis.close();
```
选择题常把 `while` 条件写成 `(data = fis.read()) != -1`，问循环次数。

**易错**：
- `FileOutputStream` 默认**覆盖**原文件内容。追加模式才不覆盖，第二个参数 `true` 必须记住。
- 流用完后必须 `close()`，否则可能数据丢失。
- `read()` 返回 `int` 而不是 `byte`，因为需要用 `-1` 表示结束，而 `byte` 范围是 -128~127，可能和合法字节值冲突。

### 考点3：文件字符流

**考试定位**：选择题判断适合读文本文件，与字节流对比。

**`FileReader`**：
```java
FileReader fr = new FileReader("test.txt");
```

**`FileWriter`**：
```java
FileWriter fw = new FileWriter("test.txt");
FileWriter fw = new FileWriter("test.txt", true);  // 追加
```

**与字节流的区别**：
- 字符流一次读写一个**字符**（2字节），适合文本文件。
- 二进制文件（如图片）必须用字节流，字符流会破坏数据。
- `FileReader` 只能使用平台默认编码，如果需要指定编码，应使用 `InputStreamReader` 类。

**读程序常考题**：
```java
FileReader fr = new FileReader("a.txt");
int ch;
while ((ch = fr.read()) != -1) {
    System.out.print((char) ch);
}
fr.close();
```
结构同字节流，`read()` 返回 `int` 的 Unicode 编码。

### 考点4：缓冲流

**考试定位**：选择题识别装饰器模式，判断效率提升方式。

**作用**：内部维护缓冲区，减少实际 I/O 次数，提高效率。

**构造方式——包装已有流**：
```java
BufferedReader br = new BufferedReader(new FileReader("test.txt"));
BufferedWriter bw = new BufferedWriter(new FileWriter("test.txt"));
BufferedInputStream bis = new BufferedInputStream(new FileInputStream("test.dat"));
BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("test.dat"));
```

**`BufferedReader` 特有方法**：
```java
String line = br.readLine();  // 读取一行文本，末尾返回 null
```

**`BufferedWriter` 特有方法**：
```java
bw.newLine();   // 写入一个行分隔符
bw.flush();     // 刷新缓冲区，强制写出
```

**易错**：
- `BufferedReader` 没有 `readLine()`？错，它有。`FileReader` 才没有。
- `newLine()` 是 `BufferedWriter` 的方法，`BufferedOutputStream` 没有。
- 缓冲流关闭时，被包装的底层流也会自动关闭。

### 考点6：对象流与序列化

**考试定位**：选择题和简答题，必考。

**序列化**：把对象的状态转换成字节序列，以便存储或传输。  
**反序列化**：从字节序列恢复对象。

**对象流**：
```java
ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("obj.dat"));
oos.writeObject(obj);
oos.close();

ObjectInputStream ois = new ObjectInputStream(new FileInputStream("obj.dat"));
Object obj = ois.readObject();
ois.close();
```

**序列化条件**：对象的类必须实现 `Serializable` 接口。

```java
class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    // ...
}
```

**`Serializable` 接口**：
- 是一个**标记接口**，没有方法需要实现。
- 未实现该接口的类对象用 `ObjectOutputStream` 写入时会抛出 `NotSerializableException`。

**`serialVersionUID`**：
- 用于版本控制。反序列化时 JVM 会比较流的 UID 和类的 UID 是否一致。
- 不显式定义的话，编译器自动生成，类改动后 UID 会变，导致已保存的对象无法反序列化。

**`transient` 关键字**：
- 修饰的字段在序列化时被忽略，反序列化后为默认值。
```java
private transient String password;  // 序列化时不保存
```

**典型选择题**：
- "`Serializable` 有方法需要实现" — 错，标记接口。
- "`transient` 修饰的变量序列化后保持原值" — 错，恢复为默认值。

### 考点11：`RandomAccessFile`（随机访问文件）

**考试定位**：选择题识别特点。

- 既不是输入流也不是输出流，**实现了 `DataInput` 和 `DataOutput` 接口**。
- 可以跳到文件的任意位置读写。
```java
RandomAccessFile raf = new RandomAccessFile("test.txt", "rw");
raf.seek(10);     // 跳到第10个字节
byte b = raf.readByte();
raf.writeBytes("hello");
raf.close();
```
- 模式：`"r"` 只读，`"rw"` 读写。

## 第十章 多线程


---

### 考点1：程序、进程与线程

| 概念 | 说明 |
|------|------|
| 程序 | 静态的代码文件，存储在硬盘上 |
| 进程 | 运行中的程序，拥有独立的内存空间 |
| 线程 | 进程内的执行单元，多个线程共享进程资源 |

**常考结论**：
- 一个进程至少有一个线程（主线程）。
- 线程是CPU调度的基本单位，进程是资源分配的基本单位。
- 多线程的意义：提高程序并发性，充分利用CPU。

你打开电脑，桌面上放着一个 `游戏.exe` 文件。这个 `.exe` 文件安安静静躺在硬盘里，它就是**程序**——静态的代码，什么也不干。

你双击它，系统把它加载到内存里，CPU 开始执行。这个正在运行的活的东西，叫**进程**。一个程序可以同时启动多次（你开了三个记事本窗口），就产生三个独立进程，各有各的内存空间，互不干扰。

每个进程内部至少包含一条执行线索，顺着代码一条条往下走——这就是**线程**。一个进程就像一个工厂，线程就是工厂里的工人。一个工厂至少有一个工人干活，也可以同时有多个工人各自干不同的事。


**判断题陷阱**：“一个进程可以包含多个进程”——错，进程包含线程，不是包含进程。

---

### 考点2：线程的创建与启动


**两种方式，必考**：

**方式一：继承 `Thread` 类**
```java
class MyThread extends Thread {
    public void run() {
	    // run 里面写的就是新线程要干的活
        System.out.println("线程运行");
    }
}
// 使用
public class Test {
    public static void main(String[] args) {
        MyThread t = new MyThread();   // 创建线程对象，此时线程处于"新建"状态
        t.start();                     // 启动线程，JVM 会自动调用 run()
        System.out.println("主线程结束");
    }
}
```

**方式二：实现 `Runnable` 接口**
```java
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("线程运行");
    }
}
// 使用
public class Test {
    public static void main(String[] args) {
        Runnable task = new MyRunnable();   // 这是任务，不是线程
        Thread t = new Thread(task);        // 把任务交给线程
        t.start();
    }
}
```

**核心陷阱**：
- 直接调用 `run()` 只是普通方法调用，不启动新线程，相当于你在主线程里执行了一遍 run。必须调用 `start()` 才创建新线程并执行 `run()`。
- `Runnable` 实现类没有 `start()` 方法，必须包装成 `Thread` 对象。
- `Thread` 本身也实现了 `Runnable`，所以可以将 `Thread` 子类对象传给另一个 `Thread` 构造器（很少这样用，但选择题可能出现）。

**匿名内部类写法（选择题常见）**：
```java
new Thread() {
    public void run() { ... }
}.start();

new Thread(new Runnable() {
    public void run() { ... }
}).start();
```

### 考点3：线程的生命周期——五个状态

**五个状态（必背）**：

``` 
新建 --start()--> 就绪 <--(拿到CPU)--> 运行 --run()结束--> 死亡
                  ^                     |
                  |   (sleep时间到)      |  (sleep/join/wait/拿
				  |						|               不到锁)
                  |                     ↓
                  +--------- 阻塞 <------+
```

**状态转换条件**（高频选择题素材）：

| 触发操作 | 状态变化 |
|----------|----------|
| `new Thread()` | 新建状态 |
| `start()` | 进入就绪队列，等待CPU |
| 获得CPU时间片 | 进入运行状态 |
| `sleep()` | 进入阻塞，时间到后回就绪 |
| `join()` | 等待另一个线程结束，阻塞 |
| `wait()` | 释放锁进入等待池，等待 `notify()` |
| `synchronized` 未获锁 | 进入锁池，阻塞 |
| `run()` 执行完毕 | 进入死亡状态 |

**注意**：`stop()` 已过时，考试可能提到但不推荐使用。`yield()` 从运行状态回到就绪，让出CPU但不阻塞。

### 考点4：线程优先级


- 优先级范围：`Thread.MIN_PRIORITY` (1) 到 `Thread.MAX_PRIORITY` (10)，默认 `NORM_PRIORITY` (5)。
- 优先级高的线程获得CPU的概率更高，但**不保证一定先执行**。
- 可继承：线程默认继承父线程的优先级。

**陷阱**：“优先级高的线程一定先执行”——错，只是概率大。

### 考点5：线程常用方法（必考读程序）

| 方法               | 作用                                                                |
| ---------------- | ----------------------------------------------------------------- |
| `sleep(long ms)` | 当前线程休眠指定毫秒，不释放锁，线程阻塞                                              |
| `yield()`        | 让出CPU，回到就绪状态                                                      |
| `join()`         | 当前线程等待调用该方法的线程死亡                                                  |
| `interrupt()`    | 给线程设置中断标志，若线程在 `sleep`/`wait`/`join` 中，会抛出 `InterruptedException` |
| `isAlive()`      | 测试线程是否存活（已启动但未死亡）                                                 |

**`sleep()` 和 `wait()` 的区别**（高频简答）：
- `sleep` 是 `Thread` 类的静态方法，任何地方都能调，调用后线程休眠但不释放对象锁，时间到了自动醒。
- `wait` 是 `Object` 类的方法，必须在同步块中调用，调用后线程释放锁并进入等待队列，靠别人调 `notify()` / `notifyAll()` 叫醒。
- **一句话**：`sleep` 是人睡着了但不放手，`wait` 是人走进等候室并把钥匙放桌上。

**读程序经典题**：
```java
class MyThread extends Thread {
    public void run() {
        System.out.print("1");
        try { Thread.sleep(1000); } catch (Exception e) { }
        System.out.print("2");
    }
}
MyThread t = new MyThread();
t.start();
System.out.print("3");
// 输出顺序：3 然后等1秒 1 2（因为主线程和子线程并发，打印3可能先于1）
// 准确说输出 3 然后 1 然后 1秒后 2，或 1 3 2 都有可能，看调度
```

### 考点6：线程同步与锁

**为什么需要同步**：多个线程同时操作同一个数据，会出乱子。

经典例子：银行取钱。账户余额 1000 元，两条线程同时取 800 元。

线程A读余额 1000 → 判断够 → 准备扣 800  
线程B读余额 1000（A还没扣完） → 判断够 → 准备扣 800

结果两条线程都取走了 800，余额变成 -600。这就是**线程不安全**。

**解决方案**：加锁，让同时只能有一个线程进入操作共享数据的代码块。

**同步方式一：同步代码块**
```java
synchronized (锁对象) {
    // 临界区
}
```

**同步方式二：同步方法**
```java
public synchronized void method() { ... }
// 实例方法的锁是 this
// 静态方法的锁是 类的 Class 对象

// 等价于 synchronized(this) 包住整个方法体
```

**关键原则**：
- 只有竞争同一个锁的线程才会互斥。
- 锁对象必须所有相关线程共享，局部变量作锁无意义。
- `synchronized` 保证可见性和原子性。

**锁是什么**

`this` 就是一把锁。每条线程进同步块之前，必须先拿到这把锁。拿到了才能进，拿不到就在外面等着，直到锁被释放。

**关键**：只有抢**同一把锁**的线程才会互相阻塞。你拿锁A，我拿锁B，各进各的，谁也挡不住谁。所以每调一次方法都 `new Object()` 当锁是没用的——每个线程拿到的是不同的锁。
**易错示例**：
```java
public void method() {
    Object lock = new Object();
    synchronized (lock) {  // 每次调用都 new 一个锁，不同线程可能拿到不同锁，锁不住
        ...
    }
}
```

### 考点7：线程通信（wait / notify / notifyAll）

### 为什么需要线程通信

同步（`synchronized`）解决了多个线程同时修改数据的混乱问题，但引入了一个新问题：

生产者线程生产一个产品，消费者线程消费一个产品。  
如果消费者发现仓库是空的，它应该**等待**生产者生产。  
如果生产者发现仓库满了，它应该**等待**消费者消费。

光有锁做不到"等待"和"通知"，必须靠 `wait()` 和 `notify()`。

|方法|作用|
|---|---|
|`wait()`|当前线程释放锁，进入等待状态，直到被 `notify()` 唤醒|
|`notify()`|唤醒**一个**正在等待这把锁的线程|
|`notifyAll()`|唤醒**所有**正在等待这把锁的线程|

`wait()`、`notify()`、`notifyAll()` 属于 `Object` 类的方法，必须在 `synchronized` 块内调用，否则抛 `IllegalMonitorStateException`。


**标准模板——生产者消费者：**
```java
class Store {
    private int product = 0;   // 仓库里的产品数量

    // 生产者调这个方法
    public synchronized void put() {
        while (product >= 1) {          // 仓库有货，不生产
            try {
                wait();                 // 释放锁，等待
            } catch (InterruptedException e) { }
        }
        product++;                       // 生产一个产品
        System.out.println("生产后库存：" + product);
        notifyAll();                     // 唤醒所有等待线程（包括消费者）
    }

    // 消费者调这个方法
    public synchronized void get() {
        while (product <= 0) {           // 仓库没货，不消费
            try {
                wait();                  // 释放锁，等待
            } catch (InterruptedException e) { }
        }
        product--;                       // 消费一个产品
        System.out.println("消费后库存：" + product);
        notifyAll();                     // 唤醒所有等待线程（包括生产者）
    }
}
```

**注意**：`wait()` 释放锁并等待，被唤醒后需要重新竞争锁。
**`wait()` 会抛 `InterruptedException`，必须 try-catch**

### 考点8：死锁

### 什么是死锁

两个或多个线程互相等待对方持有的锁，导致所有线程无法继续执行

**死锁产生的四个必要条件**：
1. **互斥**：资源同一时刻只能被一个线程持有
    
2. **请求与保持**：持有一个资源的同时去请求另一个
    
3. **不可剥夺**：资源不能被强行抢走，只能由持有者主动释放
    
4. **循环等待**：形成一条等待环路（A等B，B等A）
    

四个条件缺一不可，打破任意一个就能避免死锁。

### 避免死锁的方法

最常用：**按顺序加锁**。所有线程都先拿A再拿B，不要有的先A后B、有的先B后A。

### 考点9：守护线程（Daemon Thread）

### 概念

守护线程是**服务型线程**，专门为其他线程提供服务。  
当所有非守护线程结束，JVM 自动退出，守护线程**不管有没有执行完都强制终止**。

### 典型例子

垃圾回收器（GC）就是守护线程——你在工作的时候它在后台默默回收垃圾，你程序退出了它也没必要留着了。

### 代码写法

```java
Thread t = new MyThread();
t.setDaemon(true);   
// 将线程 `t` 设置为守护线程
//必须在 start() 之前设置
t.start();
```

**易错**：`setDaemon(true)` 必须在 `start()` 之前调用，否则抛出 `IllegalThreadStateException`。

守护线程创建的子线程默认**也是守护线程**。
