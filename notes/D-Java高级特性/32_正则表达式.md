# 正则表达式（Regular Expression）（`java.util.regex` 包）

`String` 类中有 `matches(regex)` 方法，`replaceAll()` 方法、`split()` 方法可以用正则表达式匹配

---
## 正则语法元素
### 一、自定义字符类 `[]`

**字符类**用于匹配某个位置上的**一个字符**

| 表达式            | 匹配内容                                  |
| -------------- | ------------------------------------- |
| `[]`           | 里面的字符出现一次                             |
| `[abc]`        | a、b 或 c（只匹配一个字符）                      |
| `[^abc]`       | 除 a、b、c 以外的任意一个字符（^在开头是否定）            |
| `[a-z]`        | a 到 z 之间的任意一个字符（范围）                   |
| `[a-zA-Z]`     | a到z A到Z 任意一个英文字母（大小写）                 |
| `[a-d[m-p]]`   | a到d或m到o                               |
| `[0-9]`        | 任意一个数字                                |
| `[a-z&&[def]]` | a到z和def。为：d，e，f                       |
| `[a-z&&[^bc]]` | a 到 z 中除去 b 和 c（交集与减法，`&&` 交集，`^` 排除） |

---
### 二、预定义字符类

预定义字符类是常用字符类的简写

| 表达式  | 匹配内容                             | 等价写法             |
| ---- | -------------------------------- | ---------------- |
| `.`  | 任意一个字符（默认不匹配行终止符，DOTALL 模式下匹配所有） |                  |
| `\d` | 一个数字                             | `[0-9]`          |
| `\D` | 一个非数字                            | `[^0-9]`         |
| `\w` | 一个单词字符（字母、数字、下划线）                | `[a-zA-Z0-9_]`   |
| `\W` | 一个非单词字符                          | `[^\w]`          |
| `\s` | 一个空白字符（空格、制表符、换行等）               | `[\t\n\x0B\f\r]` |
| `\S` | 一个非空白字符                          | `[^\s]`          |
| `\|` | 逻辑或                              |                  |

> Java 字符串中**必须双重转义**：写成 `"\\d"`、`"\\w"` 等，否则编译器会将 `\d` 识别为无效转义符。

### 三、数量词
| 数量词     | 匹配次数       | 说明                   |
| ------- | ---------- | -------------------- |
| `?`     | 0 或 1 次    | X?，X出现一次或者零次         |
| `*`     | 0 或多次      | X*，X出现任意次，包括零次       |
| `+`     | 1 或多次      | X+，X出现至少一次           |
| `{n}`   | 恰好 n 次     | X{n}，X出现恰好n次         |
| `{n,}`  | 至少 n 次     | X{n,}，X出现至少n次        |
| `{n,m}` | n 到 m 次（含） | X{n,m}，X出现至少n次，不超过m次 |
- **贪婪**（默认）：尽可能多匹配，能回溯。
    
- **勉强**（后缀加`?`）：尽可能少匹配，可扩展。
    
- **占有**（后缀加`+`）：尽可能多匹配，不回溯。

**例**：对字符串 `"aaaa"` 应用正则 `a*`（贪婪）匹配整个字符串；`a*?`（勉强）匹配空字符串；

### 四、边界匹配符
| 语法   | 含义                           |
| ---- | ---------------------------- |
| `^`  | 行的开头（在 `[]` 外），或取反（在 `[]` 内） |
| `$`  | 行的结尾                         |
| `\`  | 转义                           |

### 五、分组与捕获

**捕获分组**

| 语法    | 含义                            |
| ----- | ----------------------------- |
| `(X)` | 捕获组，括号中的内容会被捕获。从 1 开始，以左括号为基准 |

**非捕获分组**

| 语法           | 含义                                      |
| ------------ | --------------------------------------- |
| `(?:X)`      | 非捕获组，只分组不捕获，不分配组号                       |
| `(?=X)`      | 非捕获组，右侧必须能匹配 X                          |
| `(?!X)`      | 非捕获组，右侧不能匹配 X                           |
| `(?<name>X)` | 命名捕获组，名称 `name` 可通过 `group("name")` 获取  |
| `\n`         | 反向引用，引用编号为 `n` 的捕获组匹配到的内容（`\1`、`\2`...） |
| `\k<name>`   | 命名反向引用，引用命名组 `name` 的匹配                 |
| `$组号`        | 正则外使用，用$符                               |

### 六、模式修饰符（内嵌标志）

| 语法               | 含义                            |
| ---------------- | ----------------------------- |
| `(?i)` / `(?-i)` | 开启/关闭忽略大小写                    |
| `(?m)` / `(?-m)` | 开启/关闭多行模式                     |
| `(?s)` / `(?-s)` | 开启/关闭点号匹配换行                   |
| `(?x)` / `(?-x)` | 开启/关闭注释模式                     |

# Pattern类（`java.util.regex`）

`java.util.regex` 包下的核心类，用于 Java 中的正则操作。

**作用**：正则表达式的编译表示。一个 Pattern 对象是一个**不可变**且**线程安全**的正则规则，可以被多个 Matcher 重用。

---
**核心方法**

| 返回值              | 方法                                          | 说明                        |
| ---------------- | ------------------------------------------- | ------------------------- |
| `static Pattern` | `compile(String regex)`                     | 编译正则表达式，返回 Pattern 对象。    |
| `Matcher`        | `matcher(CharSequence input)`               | 创建匹配器，用于对 `input` 进行匹配操作。 |
| `static boolean` | `matches(String regex, CharSequence input)` | 快捷全量匹配                    |
| `String`         | `pattern()`                                 | 返回该 Pattern 所编译的正则表达式字符串。 |

---
# Matcher 类（`java.util.regex`）

`java.util.regex` 包下的核心类，用于 Java 中的正则操作。

**作用**：通过调用 Pattern 的 `matcher(CharSequence input)` 得到，用于对特定输入执行各种匹配操作。**不可变，线程不安全**。

---
**核心方法**

| 返回值        | 方法                               | 说明                                           |
| ---------- | -------------------------------- | -------------------------------------------- |
|            | **匹配操作**                         |                                              |
| `boolean`  | `matches()`                      | 尝试将**整个**输入序列与模式匹配（全量匹配）                     |
| `boolean ` | `lookingAt()`                    | 尝试从开头匹配，不要求到结尾                               |
| `boolean`  | `find()`                         | 扫描输入序列，寻找**任意位置**的下一个匹配子串（部分匹配）。重复调用会继续查找下一个 |
| `boolean`  | `find(int start)`                | 从指定索引 `start` 开始查找                           |
|            | **分组（Group）与捕获**                 |                                              |
| `String`   | `group()`                        | 返回上一次匹配操作（如 `find`）所匹配到的整个子串（等价于 `group(0)`） |
| `String`   | `group(int group)`               | 返回指定编号的捕获组内容。若组未匹配到内容，返回 `null`              |
| `String`   | `group(String name)`             | 返回命名捕获组内容                                    |
| `int`      | `groupCount()`                   | 返回捕获组的数量（不包含组 0）                             |
|            | **替换方法**                         |                                              |
| `String`   | `replaceAll(String replacement)` | 替换所有匹配                                       |

**`matches` vs `find`** 是最常见的混淆点：`matches` 需要整个字符串符合模式，`find` 只要包含即可。

---
# Pattern 与 Matcher 的关系

```java
Pattern p = Pattern.compile(regex)  →  创建 Pattern 对象
Matcher m = p.matcher(input)  →  创建 Matcher 对象
m.find() / matches()  →  执行匹配
m.group(n)        →  提取结果
```

**一个 Pattern 可以多次调用 `matcher()` 创建多个 Matcher**，同时处理不同输入。  
**Matcher 不是线程安全的**，不要跨线程共享同一个 Matcher，但 Pattern 可以。
