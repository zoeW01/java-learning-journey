
### 1. 为什么必须用命令行

图形界面只能做固定操作。后面你要启动服务器、查看日志、配置环境变量、批量操作文件——这些都没有图形按钮，只能命令行。面试时被问到“你用过哪些 Linux 命令”是大概率事件，回答不上来说明你从未接触过真实开发环境。

### 2. 文件系统结构

Linux 没有 C 盘 D 盘。所有文件和目录挂在一棵从 `/` 开始的树下。

| 路径 | 含义 | 你用它做了什么 |
|------|------|---------------|
| `/home/zoe/`（即 `~`） | 你的用户主目录 | 存放 java-projects、git-repos |
| `/mnt/c/` | Windows 的 C 盘挂载点 | 访问 Obsidian 笔记库 |
| `/usr/lib/jvm/` | JDK 安装位置 | `mvn -version` 时看到了这个路径 |

### 3. 核心命令清单

**目录操作：**

| 命令            | 作用                   | 之前用过的例子                            |
| ------------- | -------------------- | ---------------------------------- |
| `pwd`         | 显示当前所在目录的完整路径        | 当前在哪个目录                            |
| `ls`          | 列出当前目录下的文件和子目录       | `ls ~/git-repos/notes-repo/*.md`   |
| `ls -la`      | 列出所有文件（包括隐藏文件）及详细信息  | `ls -la ~/.ssh/`                   |
| `cd 目录`       | 进入某个目录               | `cd ~/java-projects/09-maven-jdbc` |
| `mkdir -p 目录` | 创建目录，`-p` 自动创建缺失的父目录 | `mkdir -p ~/git-repos/notes`       |
| `rm 文件`       | 删除文件                 | `rm -f code`                       |
| `rm -rf 目录`   | 递归强制删除整个目录           | 清理错误的 git 初始化                      |

**文件操作：**

| 命令           | 作用                | 之前用过的例子                                         |
| ------------ | ----------------- | ----------------------------------------------- |
| `nano 文件名`   | 用 nano 编辑器打开或创建文件 | `nano pom.xml`                                  |
| `cat 文件名`    | 将文件内容输出到终端        | `cat pom.xml`                                   |
| `cp 源 目标`    | 复制文件              | `cp -r ~/java-projects/07-jdbc ~/git-repos/...` |
| `cp -r 源 目标` | 递归复制整个目录          | 同上                                              |
| `echo "文字"`  | 输出文字到终端           | `echo ".obsidian/workspace.json" >> .gitignore` |
| `>> 文件名`     | 将输出追加到文件末尾        | 同上                                              |

**系统与权限：**

| 命令 | 作用 |
|------|------|
| `sudo 命令` | 以管理员权限执行命令 |
| `sudo apt update` | 更新软件源列表 |
| `sudo apt install 软件名 -y` | 安装软件，`-y` 自动确认 |
| `which 命令` | 查看命令的完整路径 |

**进程与网络：**

| 命令 | 作用 |
|------|------|
| `Ctrl+C` | 强制终止当前正在运行的命令 |
| `curl 网址` | 命令行下载网页内容或测试网络连接 |
| `wget 网址` | 下载文件 |

### 4. 路径的核心规则

- **绝对路径**：从根 `/` 开始写完整路径，比如 `/home/zoe/java-projects/09-maven-jdbc/pom.xml`
- **相对路径**：从当前目录开始写，比如 `pom.xml`（指当前目录下的 pom.xml）或 `../`（上级目录）
- **`~`**：当前用户的主目录，等同于 `/home/zoe`
- **`.`**：当前目录
- **`..`**：上一级目录
